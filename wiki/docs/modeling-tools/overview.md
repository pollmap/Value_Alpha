# 분석 도구 실전 가이드

투자 분석의 질은 **분석 프레임워크 × 도구 숙련도**에 의해 결정됩니다. 아무리 뛰어난 밸류에이션 이론을 알고 있어도, 이를 실행할 도구를 다루지 못하면 실전에서 무용지물입니다. 이 섹션에서는 현업 투자 분석가가 실제로 사용하는 도구 생태계를 체계적으로 안내합니다.

---

## 1. 투자 분석 도구 생태계 개요

현대 투자 분석은 단일 도구가 아닌 **도구 생태계(Tool Ecosystem)** 위에서 이루어집니다. 각 도구는 고유한 강점과 한계를 가지고 있으며, 상황에 따라 적절한 도구를 선택하는 것이 핵심입니다.

### 주요 도구 카테고리

| 카테고리 | 도구 | 주요 용도 |
|---------|------|----------|
| **스프레드시트** | Excel, Google Sheets | 재무 모델링, 밸류에이션, 시나리오 분석 |
| **프로그래밍** | Python, R, VBA | 데이터 분석 자동화, 백테스팅, 대량 스크리닝 |
| **데이터 터미널** | Bloomberg, Refinitiv | 실시간 데이터, 뉴스, 분석 도구 통합 |
| **한국 데이터** | DART, FnGuide, KIS정보 | 한국 기업 재무데이터, 컨센서스, 공시 |
| **시각화** | Tableau, Power BI, matplotlib | 데이터 시각화, 대시보드 |
| **협업** | Notion, Confluence, SharePoint | 리서치 노트, 팀 협업, 지식 관리 |

### 언제 어떤 도구를 쓸 것인가

- **Excel**: 개별 기업 재무 모델, 3-Statement Model, DCF, LBO, 고객/상사에게 보여줄 모델
- **Python**: 100개 이상 종목 스크리닝, 백테스팅, API 데이터 수집 자동화, 통계 분석
- **R**: 학술 연구, 고급 통계 분석, 팩터 모델링 (실무에서는 Python에 밀리는 추세)
- **Bloomberg**: 실시간 시장 데이터, 컨센서스 확인, 빠른 Comparable 분석, 뉴스 모니터링

## 3. 현업 분석가의 도구 사용 현실

### 증권사 리서치 센터

증권사 애널리스트의 주력 도구는 압도적으로 **Excel**입니다.

- **주력**: Excel (재무 모델 90% 이상)
- **보조**: Bloomberg Terminal (데이터 조회, 뉴스), FnGuide (컨센서스)
- **보고서 작성**: PowerPoint, Word
- **특이사항**: 대부분의 증권사는 자체 엑셀 템플릿을 보유. 신입 애널리스트는 첫 6개월간 엑셀 모델링을 집중 훈련
- **Python 활용**: 퀀트 리서치 팀 중심으로 확대 중이나, 전통 섹터 애널리스트는 Excel 위주

### 자산운용사

- **주력**: Excel + Bloomberg (투자 의사결정 프로세스)
- **보조**: Python (스크리닝, 포트폴리오 분석), 자체 시스템
- **특이사항**: 대형 운용사는 자체 투자 관리 시스템(OMS/PMS)을 보유. Bloomberg PORT 기능으로 포트폴리오 분석
- **리스크 관리**: RiskMetrics, MSCI Barra 등 전문 리스크 시스템 병행

### IB (투자은행)

- **주력**: Excel (재무 모델, 밸류에이션), PowerPoint (Pitch Book)
- **보조**: Bloomberg, Capital IQ, FactSet
- **특이사항**: M&A 딜에서 사용하는 모델은 극도로 정교. 색상 코딩(Color Convention)을 엄격히 준수
- **면접 시 Excel 테스트**: IB 입사 시 엑셀 모델링 테스트는 필수. Paper LBO 등 수기 계산도 요구

### PE (사모펀드) / VC

- **주력**: Excel (LBO 모델, Returns Analysis)
- **보조**: Capital IQ, PitchBook
- **특이사항**: LBO 모델은 IB보다 더 정교. Operating Model을 별도 탭으로 구축하는 경우가 많음

## 4. 학습 로드맵

### 초보자 단계 (0~3개월)

**목표**: Excel 기본 + 재무제표 읽기

| 주차 | 학습 내용 | 실습 |
|------|----------|------|
| 1-2주 | Excel 기본 함수 (SUM, IF, VLOOKUP) | 간단한 손익 테이블 만들기 |
| 3-4주 | 재무제표 구조 이해 | DART에서 삼성전자 재무제표 다운로드 |
| 5-6주 | Excel 중급 함수 (INDEX/MATCH, SUMPRODUCT) | 재무비율 자동 계산 시트 구축 |
| 7-8주 | 차트/그래프 작성 | 5년치 실적 트렌드 시각화 |
| 9-12주 | 간단한 밸류에이션 (PER/PBR 비교) | 동종업종 3개사 비교 분석 |

### 중급 단계 (3~9개월)

**목표**: 3-Statement Model 독립 구축 + Python 기초

| 주차 | 학습 내용 | 실습 |
|------|----------|------|
| 1-4주 | 3-Statement Model 구축 (IS → BS → CF) | 실제 기업 모델 구축 (삼성전자 or LG화학) |
| 5-8주 | DCF 밸류에이션 | WACC 계산, Terminal Value 산출, Football Field 차트 |
| 9-12주 | Python 기초 (pandas, numpy) | Jupyter Notebook으로 주가 데이터 분석 |
| 13-16주 | Python 데이터 수집 (yfinance, DART API) | 자동 재무비율 계산 스크립트 |
| 17-20주 | 시나리오/민감도 분석 | Excel 데이터 테이블 + Python 몬테카를로 |
| 21-24주 | 리서치 보고서 작성 | 완성된 모델 기반 투자 의견 보고서 |

### 고급 단계 (9개월~)

**목표**: 복잡한 모델 구축 + 자동화

| 주차 | 학습 내용 | 실습 |
|------|----------|------|
| 1-8주 | LBO 모델링 | PE 스타일 LBO 모델 구축 |
| 9-12주 | SOTP (Sum-of-the-Parts) 밸류에이션 | 지주회사/복합기업 밸류에이션 |
| 13-16주 | Python 포트폴리오 분석 | 효율적 프론티어, 팩터 분석 |
| 17-20주 | VBA/매크로 자동화 | 데이터 업데이트 자동화 매크로 |
| 21-24주 | Bloomberg/API 연동 | Excel-Bloomberg 연동, Python-API 파이프라인 |
| 25주+ | 산업별 특수 모델링 | 은행 모델(NIM), 보험 모델(EV), 부동산(NAV) |

## 5. 한국 시장 특화 도구

### DART OpenAPI

[DART(전자공시시스템)](https://dart.fss.or.kr)는 한국 상장기업의 모든 공시를 열람할 수 있는 금융감독원 시스템입니다.

- **API 키 발급**: dart.fss.or.kr → 개발자센터 → API 키 신청 (무료)
- **주요 기능**: 재무제표 조회, 공시 검색, 대량 다운로드
- **일일 호출 한도**: 10,000건 (개인), 상위 플랜 별도 문의
- **Python 라이브러리**: `OpenDartReader` (추천), `dart-fss`

### FnGuide

국내 최대 금융 데이터 제공업체로, 증권사/운용사에서 가장 많이 사용합니다.

- **DataGuide Pro**: 엑셀 Add-in 방식, 재무 데이터/컨센서스/밸류에이션 데이터 제공
- **FnGuide 웹**: 기업 개요, 재무제표, 컨센서스, 주요 주주 정보
- **비용**: 기관 전용 유료 서비스 (개인은 제한적 웹 접근 가능)
- **대안**: 네이버 금융에서 FnGuide 데이터 일부 무료 제공

### KIS정보 (한국투자증권 정보)

- **KIS-VALUE**: 기업 재무 데이터베이스, 학계에서 많이 사용
- **NICE평가정보**: 신용등급, 비상장기업 재무 데이터
- **용도**: 학술 연구, 대량 데이터 분석

### 네이버 금융

개인 투자자가 가장 쉽게 접근할 수 있는 무료 데이터 소스입니다.

- **주요 기능**: 실시간 시세, 재무제표 요약, 컨센서스, 뉴스
- **한계**: 과거 데이터 깊이 제한, API 미제공 (크롤링 시 이용약관 주의)
- **팁**: 종목 분석 탭의 "투자의견 컨센서스"는 FnGuide 데이터 기반

### KRX (한국거래소)

- **KRX 정보데이터시스템** (data.krx.co.kr): 시장 통계, 투자자별 매매 동향, 공매도 잔고
- **무료 다운로드**: CSV 형태로 과거 데이터 다운로드 가능
- **API**: 공식 API는 없으나, Python `FinanceDataReader` 라이브러리로 접근 가능

## 6. 도구 선택 의사결정 프레임워크

분석 과제가 주어졌을 때, 다음 질문을 통해 적절한 도구를 선택하세요.

```
[시작]
│
├─ 분석 대상이 1~3개 기업인가? ──Yes──▶ Excel 재무 모델
│                                         │
├─ 대상이 10개 이상인가? ──Yes──▶ Python 스크리닝
│                                    │
├─ 실시간 데이터가 필요한가? ──Yes──▶ Bloomberg / HTS
│                                        │
├─ 통계적 검증이 필요한가? ──Yes──▶ Python or R
│                                      │
├─ 결과를 상사/고객에게 보여줘야 하는가? ──Yes──▶ Excel + PPT
│                                                    │
└─ 반복 작업인가? ──Yes──▶ Python 자동화 or VBA 매크로
```

각 페이지는 이론보다 **실습과 실전**에 초점을 맞추었습니다. 코드와 엑셀 수식을 직접 따라 하면서 학습하시기 바랍니다.

:::tip 학습 순서 추천
1. 먼저 **엑셀 재무 모델링**을 완주하세요. 현업의 95%는 여전히 Excel 기반입니다.
2. 기본 모델링이 가능해지면 **파이썬 금융 분석**으로 자동화와 대량 분석 역량을 확장하세요.
3. 마지막으로 **현업 분석 방법론**을 통해 프로 분석가의 워크플로우를 이해하세요.
:::

:::caution 도구는 수단일 뿐
아무리 도구를 잘 다뤄도 기업과 산업에 대한 이해가 없으면 의미 있는 분석이 불가능합니다. [투자분석 체계](/docs/foundation/overview)의 기초를 반드시 먼저 학습하세요.
:::
