# 엑셀 재무 모델링 완전 정복

엑셀은 금융업계의 공용어(lingua franca)입니다. Goldman Sachs, Morgan Stanley 등 글로벌 IB부터 국내 증권사, 자산운용사까지 거의 모든 재무 모델은 Excel 위에서 만들어집니다. 이 가이드는 신입 애널리스트 수준에서 시니어 수준까지 엑셀 재무 모델링의 모든 것을 다룹니다.

### 1.2 색상 규칙 (Color Convention)

IB/증권사에서 공통으로 사용하는 표준 색상 규칙입니다. 이를 따르면 모델의 가독성이 크게 향상됩니다.

- **파란색 (Blue)** — 용도: 하드코딩 입력값, 의미: 사용자가 직접 입력하는 가정(assumption), 예시: 매출 성장률 5%, 할인율 10%
- **검은색 (Black)** — 용도: 수식/계산값, 의미: 다른 셀을 참조하여 자동 계산되는 값, 예시: `=B5*B6`, `=SUM(B5:B10)`
- **녹색 (Green)** — 용도: 다른 시트/파일 링크, 의미: 다른 워크시트나 외부 파일에서 가져온 값, 예시: `=IS!B5`, `='[다른파일.xlsx]Sheet1'!A1`
- **빨간색 (Red)** — 용도: 경고/체크, 의미: 밸런스 체크 실패, 에러 표시, 예시: BS 불일치 경고, 음수 현금

### 1.3 시트 구조 표준

```
[Cover] → [Assumptions] → [IS] → [BS] → [CF] → [Debt Schedule] →
[Depreciation] → [Working Capital] → [DCF] → [Comps] → [Sensitivity] → [Output]
```

- **Cover**: 기업명, 분석일, 분석가, 버전 정보
- **Assumptions**: 모든 주요 가정을 한 곳에 모아둠 (파란색 입력)
- **IS/BS/CF**: 과거 실적 + 추정치 (과거=검정, 추정=파란)
- **Output**: 최종 밸류에이션 결과 요약

## 2. 3-Statement Model 구축 완전 가이드

3-Statement Model은 손익계산서(IS), 재무상태표(BS), 현금흐름표(CF)를 하나의 통합 모델로 연결하는 재무 모델링의 기본입니다.

### Step 1: 과거 재무제표 입력

#### DART에서 재무제표 가져오기

1. [DART](https://dart.fss.or.kr) 접속 → 기업 검색 (예: 삼성전자)
2. 사업보고서 → 재무제표 → "연결재무제표" 선택
3. 엑셀 다운로드 클릭 (XBRL 뷰어 또는 직접 복사)
4. 최소 3년, 가능하면 5년치 과거 데이터 입력

#### 데이터 정리 원칙

- **단위 통일**: 억원 또는 백만원 (모델 상단에 명시)
- **기간 통일**: 연간(Annual) 기준, 필요 시 분기(Quarterly) 별도 시트
- **항목 표준화**: DART 원본 항목명을 분석용 표준 항목으로 매핑

```
DART 원본                    →  모델 표준 항목
─────────────────────────────────────────────
매출액                       →  Revenue (매출)
매출원가                     →  COGS (매출원가)
판매비와관리비               →  SG&A (판관비)
영업이익                     →  Operating Income (영업이익)
금융수익/비용                →  Net Interest (순금융비용)
법인세비용                   →  Tax (법인세)
당기순이익                   →  Net Income (당기순이익)
```

### Step 2: 매출 드라이버 모델링

#### Top-Down 방식

시장 전체 규모에서 출발하여 기업의 점유율을 곱하는 방식입니다.

```
시장 규모(TAM) × 시장 점유율 = 매출
```

**엑셀 구현 예시:**

- **글로벌 반도체 시장 (십억$)** — FY2022: 573, FY2023: 527, FY2024E: 588, FY2025E: 650
- **메모리 비중 (%)** — FY2022: 25%, FY2023: 22%, FY2024E: 24%, FY2025E: 26%
- **삼성전자 메모리 M/S (%)** — FY2022: 42%, FY2023: 40%, FY2024E: 41%, FY2025E: 42%
- **메모리 매출 (십억$)** — FY2022: **=B2×B3×B4**, FY2023: ..., FY2024E: ..., FY2025E: ...

#### Bottom-Up 방식

제품/사업부별로 수량 × 단가를 적산하는 방식입니다.

```
Σ (제품별 판매 수량 × 제품별 평균 판매 단가) = 매출
```

**엑셀 구현 예시:**

- **DS (반도체)** — 항목: 출하량 (백만개), FY2023: 150, FY2024E: 170, FY2025E: 190
- 항목: ASP ($), FY2023: 3.2, FY2024E: 3.8, FY2025E: 4.0
- 항목: **매출**, FY2023: **=B3×B4**, FY2024E: ..., FY2025E: ...
- **MX (스마트폰)** — 항목: 출하량 (백만대), FY2023: 225, FY2024E: 230, FY2025E: 235
- 항목: ASP (만원), FY2023: 45, FY2024E: 47, FY2025E: 48
- 항목: **매출**, FY2023: **=B7×B8**, FY2024E: ..., FY2025E: ...

### Step 3: 비용 구조 모델링

비용을 변동비, 고정비, 반고정비로 분류하여 모델링합니다.

#### 변동비 (Variable Cost)

매출에 비례하여 변동하는 비용. **매출 대비 비율(% of Revenue)**로 추정합니다.

```
매출원가 = 매출 × 매출원가율
  예: =Revenue * COGS_Margin

직접인건비, 원재료비, 물류비 등
```

#### 고정비 (Fixed Cost)

매출과 무관하게 일정한 비용. **전년 대비 증가율** 또는 **절대 금액**으로 추정합니다.

```
임차료 = 전기 임차료 × (1 + 상승률)
감가상각비 = 별도 감가상각 스케줄에서 산출
```

#### 반고정비 (Semi-Variable Cost)

매출 규모에 따라 단계적으로 변하는 비용. 과거 추세와 판단을 결합하여 추정합니다.

```
인건비 = 직원 수 × 1인당 평균 급여
R&D = 매출 대비 비율 (단, 최소 금액 존재)
```

### Step 4: 운전자본 모델링 (Working Capital)

운전자본(NWC) 변동은 현금흐름에 직접적인 영향을 미칩니다.

#### 핵심 지표

```
DSO (매출채권 회수일) = 매출채권 / 매출 × 365
DIO (재고자산 회전일) = 재고자산 / 매출원가 × 365
DPO (매입채무 지급일) = 매입채무 / 매출원가 × 365
CCC (현금전환주기)    = DSO + DIO - DPO
```

#### 엑셀 수식 예시

```
매출채권(E) = 매출(E) / 365 × DSO_가정
재고자산(E) = COGS(E) / 365 × DIO_가정
매입채무(E) = COGS(E) / 365 × DPO_가정

NWC = 매출채권 + 재고자산 - 매입채무
ΔNWC = NWC(당기) - NWC(전기)   ← CF에 반영 (증가 시 현금 유출)
```

### Step 5: CAPEX / 감가상각 스케줄

#### 감가상각 스케줄 구조

- **기초 유형자산** — FY2022: 100,000, FY2023: 110,000, FY2024E: 118,000, FY2025E: 126,000
- **(+) CAPEX** — FY2022: 20,000, FY2023: 18,000, FY2024E: 20,000, FY2025E: 22,000
- **(-) 감가상각비** — FY2022: (10,000), FY2023: (10,000), FY2024E: (12,000), FY2025E: (13,000)
- **(-) 처분/폐기** — FY2022: 0, FY2023: 0, FY2024E: 0, FY2025E: 0
- **기말 유형자산** — FY2022: 110,000, FY2023: 118,000, FY2024E: 126,000, FY2025E: 135,000

#### CAPEX 추정 방법

- **매출 대비 비율**: CAPEX/매출 비율의 과거 평균을 적용 (가장 보편적)
- **유지보수 vs 성장**: Maintenance CAPEX (감가상각 수준) + Growth CAPEX (별도 추정)
- **경영진 가이던스**: IR 자료에서 CAPEX 계획 확인 (가장 정확)

```
CAPEX = 매출 × CAPEX_to_Revenue_비율
  또는
CAPEX = Maintenance_CAPEX + Growth_CAPEX
  여기서 Maintenance_CAPEX ≈ 감가상각비
```

### Step 7: 3표 연결 (Three-Statement Linkage)

이것이 재무 모델링의 핵심입니다. IS, BS, CF가 올바르게 연결되어야 합니다.

#### 연결 로직

```
[손익계산서 → 현금흐름표]
  당기순이익 → CF 영업활동 시작점

[현금흐름표 → 재무상태표]
  기초 현금 + 영업CF + 투자CF + 재무CF = 기말 현금 → BS 현금 항목

[재무상태표 → 손익계산서]
  기초 차입금 → 이자비용 계산 → IS 이자비용
  기초 유형자산 → 감가상각비 → IS 비용 항목
```

#### 순환참조(Circular Reference) 처리

3표 모델에서 순환참조는 자연스럽게 발생합니다.

**발생 원인:**
```
이자비용 → 당기순이익 → 현금흐름 → 기말 현금 → 차입금 필요액 → 이자비용 (순환!)
```

**해결 방법 1: Iterative Calculation 설정**

```
파일 → 옵션 → 수식 → "반복 계산 사용" 체크
  최대 반복 횟수: 100
  최대 변경값: 0.001
```

**해결 방법 2: Circuit Breaker 방식 (권장)**

Assumptions 시트에 Circuit Breaker 셀을 만들어 순환참조를 on/off 합니다.

```
셀 A1 (Assumptions 시트): "Circular ON/OFF" → 1 또는 0 (파란색)

이자비용 수식:
=IF(CircularSwitch=1, Average_Debt * Interest_Rate, 이전기_이자비용)
```

이렇게 하면 모델이 꼬였을 때 CircularSwitch를 0으로 바꿔 순환참조를 끊고 디버깅할 수 있습니다.

### Step 8: DCF 밸류에이션 연결

3-Statement Model에서 DCF로 연결하는 과정입니다.

#### FCFF (Free Cash Flow to Firm) 산출

```
FCFF = EBIT × (1 - 세율)
     + 감가상각비
     - CAPEX
     - ΔNWC

각 항목은 IS, BS, CF 시트에서 링크:
  EBIT        → =IS!Operating_Income
  세율         → =IS!Tax / IS!EBT  (유효세율)
  감가상각비    → =CF!Depreciation  또는 =Depr_Schedule!총감가상각
  CAPEX       → =CF!CAPEX
  ΔNWC        → =WC_Schedule!Delta_NWC
```

#### Terminal Value 계산

```
TV (영구성장률 방식) = FCF(최종연도) × (1 + g) / (WACC - g)
TV (Exit Multiple 방식) = EBITDA(최종연도) × Exit_EV/EBITDA
```

#### 기업가치 산출

```
Enterprise Value = Σ [FCFt / (1+WACC)^t] + TV / (1+WACC)^n
Equity Value = Enterprise Value - 순차입금
주당 가치 = Equity Value / 발행주식수
```

### Step 9: 민감도 분석 (Data Table)

#### 2-Way Data Table 활용

## 4. 실전 팁

### 4.1 순환참조 해결 체크리스트

순환참조가 발생했을 때 다음 순서로 디버깅합니다:

1. **수식 → 오류 추적 → 순환 참조** 메뉴에서 순환 셀 위치 확인
2. `Ctrl + ~`로 수식 표시 모드 전환하여 로직 추적
3. Circuit Breaker를 0으로 설정하여 순환 끊기
4. 각 연결 포인트를 하나씩 끊어보며 원인 파악
5. 순환 로직이 맞는지 확인 후 Iterative Calculation 재설정

### 4.2 감사 추적 (Audit Trail)

- **버전 관리**: 파일명에 날짜 포함 (예: `삼성전자_Model_v3_20241215.xlsx`)
- **변경 이력 시트**: 주요 가정 변경 시 날짜, 변경 내용, 사유 기록
- **수식 감사 도구**: `수식` 탭 → `참조 셀 추적` / `종속 셀 추적`으로 데이터 흐름 시각화
- **셀 메모 활용**: 주요 가정에 근거 자료 출처를 메모로 남김

### 4.3 모델 검증 체크리스트

모델 완성 후 반드시 아래 항목을 검증합니다:

```
□ 대차대조표 밸런스 체크: 자산 = 부채 + 자본 (모든 연도)
□ 현금흐름 검증: 기초현금 + 총CF = 기말현금
□ 매출 성장률 합리성: 산업 성장률 대비 과도하지 않은지
□ 마진 추세: 급격한 변동 없이 논리적으로 연결되는지
□ CAPEX/감가상각 관계: CAPEX > 감가상각 (성장 기업일 경우)
□ 순차입금 추이: 부채 비율이 급격히 변하지 않는지
□ 단위 확인: 원 vs 천원 vs 백만원 vs 억원 혼용 없는지
□ 참조 무결성: #REF! 에러 없는지 전체 시트 검색
□ 하드코딩 색상: 모든 가정이 파란색으로 표시되어 있는지
□ 극단값 테스트: 가정을 극단적으로 바꿔도 모델이 논리적으로 작동하는지
```

### 4.4 흔한 실수와 방지법

- **연결 vs 별도 재무제표 혼용** — 결과: 비교 불가능한 수치, 방지법: 시트 상단에 "연결/별도" 명시
- **단위 불일치** — 결과: 밸류에이션 10배~100배 오류, 방지법: 모델 상단에 단위 고정 (억원)
- **과거 데이터 오입력** — 결과: 추정 기반 자체가 틀림, 방지법: DART 원본과 교차 검증
- **수식 깨짐 (행/열 삽입 후)** — 결과: 잘못된 참조, 방지법: 이름 정의(Named Range) 활용
- **주식수 미반영** — 결과: 주당 가치 오류, 방지법: 자기주식, 전환사채 고려

## 5. 한국 실전 특화

### 5.1 DART에서 재무제표 다운로드 상세 절차

1. **dart.fss.or.kr** 접속
2. 상단 검색창에 기업명 입력 (예: "삼성전자")
3. 좌측 메뉴에서 **정기공시** → **사업보고서** 선택
4. 최근 사업보고서 클릭 → 목차에서 **"III. 재무에 관한 사항"** 클릭
5. **연결재무제표** 또는 **재무제표** 확인
6. 표를 직접 선택 → 복사 → 엑셀에 붙여넣기
7. 대안: **DART Open API**를 통해 XBRL 데이터 자동 수집 (Python 섹션 참조)

### 5.2 K-IFRS 특수 항목 처리

#### 리스 (IFRS 16)

K-IFRS 제1116호 적용 이후, 운용리스도 사용권자산/리스부채로 자본화됩니다.

```
모델링 시 주의사항:
1. 리스부채는 차입금에 포함할지 여부 결정 (EV 계산 시 영향)
2. EBITDA 비교 시 리스 적용 전/후 차이 인식
3. 항공, 유통 등 리스 비중이 큰 산업에서 특히 중요
```

#### 진행률 수익인식 (건설/조선)

```
매출 = 총 계약금액 × 진행률
진행률 = 누적발생원가 / 총추정원가

모델링 시:
  - 수주잔고(Backlog) 추적이 핵심
  - 신규 수주 추정 → 매출 인식 시점 추정
  - 원가율 변동 위험 (적자 전환 가능성)
```

### 5.3 연결 vs 별도 재무제표 선택 기준

- **분석 대상** — 연결: 그룹 전체 (자회사 포함), 별도: 모회사 단독
- **권장 상황** — 연결: 일반적인 기업 분석, DCF, 별도: 지주회사 분석, 자회사 가치 분리
- **주의사항** — 연결: 비지배지분 처리, 별도: 자회사 실적 미반영
- **밸류에이션** — 연결: EV, P/E 등 직접 적용, 별도: NAV (순자산가치) 방식

**실무 원칙**: 특별한 사유가 없으면 **연결재무제표**를 기본으로 사용합니다. 지주회사(예: SK, LG)처럼 자회사 가치 분리가 필요한 경우에만 별도재무제표를 병행합니다.

## 6. 모델 구축 실전 체크리스트
