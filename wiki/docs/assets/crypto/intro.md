# 암호화폐 가치평가

## 암호화폐의 특징

### 전통 자산과의 비교

| 특성 | 암호화폐 | 주식 | 금 |
|------|---------|------|-----|
| 내재가치 | 논쟁적 | 기업가치 | 실물가치 |
| 현금흐름 | 일부 (스테이킹) | 배당 | 없음 |
| 공급 | 프로토콜 정의 | 발행 | 채굴량 |
| 규제 | 불확실 | 명확 | 명확 |
| 거래 시간 | 24/7 | 장중 | 장중 |
| 변동성 | 매우 높음 | 중간 | 낮음 |

### 암호화폐 분류

```
1. 결제형 (Currency)
   └── Bitcoin, Litecoin, Bitcoin Cash

2. 플랫폼형 (Platform)
   └── Ethereum, Solana, Cardano

3. 유틸리티 토큰 (Utility)
   └── Chainlink, Filecoin

4. 거버넌스 토큰 (Governance)
   └── UNI, AAVE, MKR

5. 스테이블코인 (Stablecoin)
   └── USDT, USDC, DAI
```

## 가치평가 프레임워크

### 1. 네트워크 가치 모델

**메트칼프의 법칙 (Metcalfe's Law):**

```
네트워크 가치 ∝ n²

n = 활성 사용자 수

NVT (Network Value to Transactions):
NVT = 시가총액 / 일일 거래량

낮은 NVT = 활발한 사용, 저평가 가능
높은 NVT = 투기적, 고평가 가능
```

### 2. 수요-공급 모델

**Stock-to-Flow (S2F) - 비트코인:**

```
S2F = 현재 공급량 / 연간 신규 공급량

비트코인 반감기 후:
S2F 증가 → 희소성 증가 → 가격 상승 기대

주의: S2F 모델은 논쟁적
- 수요를 고려하지 않음
- 과거 상관관계 ≠ 인과관계
```

### 3. DCF 적용 (수익 창출 프로토콜)

```
이더리움 예시:

수익원:
├── 트랜잭션 수수료 (일부 소각)
├── MEV 수익
└── 스테이킹 수익

밸류에이션:
프로토콜 가치 = 미래 수수료의 현재가치

할인율 결정:
- 높은 위험 → 높은 할인율 (30-50%+)
- 규제 불확실성 반영
```

### 4. 비교 가치평가

```
유사 프로토콜 비교:

| 지표 | 프로젝트 A | 프로젝트 B |
|------|----------|----------|
| 시가총액 | $10B | $5B |
| TVL | $5B | $4B |
| 일일 사용자 | 100K | 80K |
| MC/TVL | 2.0x | 1.25x |
| 수수료 수익 | $1M/일 | $500K/일 |
| P/S | 27x | 27x |

→ 프로젝트 B가 TVL 대비 저평가?
```

## 온체인 분석 지표

### 활동 지표

```
활성 주소 (Active Addresses):
- 일일 활성 주소 수
- 추세: 증가 → 채택 확대

트랜잭션 수 (Transactions):
- 네트워크 활용도
- 수수료 수익과 연계

TVL (Total Value Locked):
- DeFi 프로토콜 규모
- 프로토콜 신뢰도 지표
```

### 보유 지표

```
HODL Waves:
- 장기 보유자 비율
- 높을수록 가격 지지력

거래소 유출입:
- 유입 증가 → 매도 압력
- 유출 증가 → HODL 의지

고래 움직임:
- 대량 보유자 동향
- 가격에 큰 영향
```

### 채굴/스테이킹 지표

```
해시레이트 (PoW):
- 네트워크 보안 수준
- 채굴자 참여도

스테이킹 비율 (PoS):
- 스테이킹된 토큰 비율
- 높을수록 매도 압력 낮음

검증자 수:
- 탈중앙화 정도
- 네트워크 건전성
```

## 토큰 경제학 (Tokenomics)

### 공급 구조 분석

```
총 공급량 (Total Supply)
├── 순환 공급량 (Circulating)
│   └── 시장에서 거래 가능
├── 락업 물량 (Locked)
│   ├── 팀 물량
│   ├── 투자자 물량
│   └── 재단 물량
└── 미발행 (Not Minted)
    └── 채굴/스테이킹 보상

핵심 체크:
□ 인플레이션율
□ 락업 해제 일정
□ 토큰 소각 메커니즘
```

### 수요 동인 분석

```
유틸리티 수요:
├── 수수료 지불
├── 거버넌스 참여
├── 스테이킹 보상
└── 서비스 접근

투기 수요:
├── 가격 상승 기대
├── 에어드롭 기대
└── 시장 심리
```

### 토큰 분배

```
건강한 분배 예시:
├── 커뮤니티: 40%+
├── 팀: 15-20% (4년 베스팅)
├── 투자자: 15-20% (2년 베스팅)
└── 재단/생태계: 20%

위험 신호:
- 팀/투자자 비율 > 50%
- 짧은 베스팅 기간
- 불투명한 분배
```

## 투자 체크리스트

### 기본 분석

```
프로젝트 평가:
□ 해결하는 문제가 실재하는가?
□ 블록체인이 필요한 이유는?
□ 경쟁 우위는 무엇인가?
□ 팀의 역량과 이력은?
□ 로드맵 달성률은?

토큰 평가:
□ 토큰의 유틸리티는?
□ 토큰 없이 프로젝트가 작동하나?
□ 공급 일정은 투명한가?
□ 인플레이션율은 적정한가?
```

### 리스크 평가

```
기술 리스크:
□ 코드 감사 여부
□ 해킹 이력
□ 스마트컨트랙트 복잡도

규제 리스크:
□ 증권성 여부
□ 주요국 규제 동향
□ 거래소 상장폐지 가능성

시장 리스크:
□ 유동성 수준
□ 거래소 집중도
□ 변동성 수준
```

## 가치평가 한계

### 주의사항

```
1. 내재가치 정의 어려움
   - 현금흐름 없는 자산 다수
   - 네트워크 효과 측정 어려움

2. 높은 불확실성
   - 기술 변화 빠름
   - 규제 환경 불확실
   - 경쟁 치열

3. 시장 비효율성
   - 정보 비대칭
   - 조작 가능성
   - 24시간 거래 피로

4. 과거 데이터 한계
   - 역사 짧음
   - 구조적 변화 빈번
```
