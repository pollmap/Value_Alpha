# 레이어2 & 확장성 솔루션

블록체인의 "트릴레마"란 보안성(Security), 탈중앙화(Decentralization), 확장성(Scalability) 세 가지를 동시에 달성하기 어렵다는 개념입니다. 이더리움은 보안성과 탈중앙화를 우선시하면서, 확장성 문제를 레이어2(Layer 2) 솔루션으로 해결하는 "롤업 중심 로드맵(Rollup-centric Roadmap)"을 채택했습니다. 이 가이드에서는 다양한 확장성 솔루션의 기술 원리, 장단점, 그리고 투자 관점에서의 분석을 상세히 다룹니다.

---

## 확장성 솔루션의 분류

### 레이어1 확장

레이어1(L1)은 블록체인 메인넷 자체의 확장을 의미합니다.

- **블록 크기 증가** — 비트코인 캐시(BCH)의 접근법. 단순하지만 노드 운영 부담 증가로 탈중앙화가 약화됨
- **합의 알고리즘 개선** — 솔라나의 PoH(Proof of History), 아발란체의 서브넷 등
- **샤딩(Sharding)** — 이더리움이 장기적으로 추진 중인 방식으로, 네트워크를 여러 조각(샤드)으로 나누어 병렬 처리. 현재는 Danksharding 방향으로 발전 중

### 레이어2 확장

레이어1 위에 구축되어 트랜잭션을 처리하되, 최종 보안은 레이어1에 의존하는 방식입니다.

**레이어2의 핵심 원리**: 트랜잭션 실행은 L2에서 하되, 데이터나 증명을 L1에 게시하여 L1 수준의 보안을 상속받습니다.

## 롤업 (Rollups)

롤업은 수백~수천 개의 트랜잭션을 L2에서 실행한 후, 압축된 데이터와 유효성 증명을 L1에 게시하는 확장 기술입니다. 현재 이더리움 확장의 주류 방식입니다.

### Optimistic Rollup

**작동 원리**: 트랜잭션이 기본적으로 유효하다고 "낙관적(optimistic)"으로 가정하고 L1에 게시합니다. 누구든 일정 기간(챌린지 기간, 보통 7일) 내에 사기 증명(Fraud Proof)을 제출하여 잘못된 트랜잭션에 이의를 제기할 수 있습니다.

**장점**:
- EVM 호환성이 높아 기존 이더리움 dApp을 쉽게 이전 가능
- 비교적 단순한 기술 구현
- 검증 비용이 낮음 (분쟁이 발생할 때만 비용 발생)

**단점**:
- 출금 시 7일의 챌린지 기간 대기 필요 (패스트 브릿지로 완화 가능)
- 사기 증명 시스템의 안전성이 중요
- 데이터 가용성에 L1 의존

**주요 프로젝트**:

| 프로젝트 | TVL | 특징 | 토큰 |
|---------|-----|------|------|
| Arbitrum One | $10B+ | 최대 L2, Nitro 기술스택, Stylus(Rust 지원) | ARB |
| Optimism (OP Mainnet) | $7B+ | OP Stack 오픈소스, Superchain 비전 | OP |
| Base | $5B+ | Coinbase 주도, OP Stack 기반, 토큰 없음 | - |

### ZK Rollup (Zero-Knowledge Rollup)

**작동 원리**: 영지식 증명(Zero-Knowledge Proof)을 사용하여 트랜잭션의 유효성을 수학적으로 증명합니다. L1에 증명을 게시하면 즉시 검증되므로 출금 대기 시간이 필요 없습니다.

**장점**:
- 수학적으로 검증 가능한 보안 (사기 증명보다 강력)
- 빠른 출금 (챌린지 기간 불필요)
- 장기적으로 더 높은 확장성 잠재력
- 프라이버시 기능 구현 가능

**단점**:
- 증명 생성에 높은 연산 비용
- EVM 완전 호환 구현이 어려움 (zkEVM의 다양한 호환성 수준)
- 기술적 복잡도가 높아 개발 속도가 느림

**주요 프로젝트**:

| 프로젝트 | 특징 | zkEVM 타입 | 토큰 |
|---------|------|-----------|------|
| zkSync Era | Matter Labs 개발, 네이티브 계정 추상화 | Type 4 | ZK |
| StarkNet | StarkWare 개발, Cairo 언어, STARK 증명 | Type 4 (커스텀) | STRK |
| Polygon zkEVM | Polygon Labs, Type 2 zkEVM 목표 | Type 2~3 | POL |
| Scroll | 이더리움 재단과 협력, 바이트코드 호환 | Type 2~3 | SCR |
| Linea | ConsenSys 개발 | Type 2 | - |

### zkEVM 호환성 등급

Vitalik Buterin이 제안한 zkEVM 분류 체계입니다.

- **Type 1** — 이더리움과 완전 동일 (현재 실현 어려움, 증명 시간이 매우 길음)
- **Type 2** — EVM 동일하지만 일부 자료구조 다름 (대부분의 dApp 호환)
- **Type 2.5** — 가스 비용이 일부 다름
- **Type 3** — 대부분 EVM 호환이나 일부 기능 미지원
- **Type 4** — 고수준 언어(Solidity)만 호환, 바이트코드 수준은 다름

:::tip zkEVM 선택 기준
개발자라면 기존 컨트랙트를 수정 없이 배포하려면 Type 2~3을, 최고의 성능을 원하면 Type 4를 선택하세요. 사용자라면 zkEVM 타입보다 생태계 규모, TVL, 사용 가능한 dApp을 기준으로 선택하는 것이 실용적입니다.
:::

## 사이드체인 (Sidechains)

사이드체인은 메인체인과 별도의 합의 메커니즘을 가진 독립적인 블록체인으로, 브릿지를 통해 메인체인과 자산을 이동합니다.

**Polygon PoS (구 Matic)**:

- 독자적인 PoS 합의 메커니즘
- 이더리움 체크포인트를 주기적으로 게시
- 낮은 가스비($0.01 미만)와 빠른 트랜잭션(2초)
- 엄밀히 L2가 아닌 사이드체인이지만, 이더리움 생태계의 핵심 확장 솔루션
- Polygon 2.0 로드맵으로 ZK 기반 L2로 전환 중 (AggLayer)

:::caution 사이드체인 vs 롤업 보안 차이
사이드체인은 자체 검증자 세트에 의존하므로 이더리움의 보안을 직접 상속받지 않습니다. 검증자들이 공모하면 자금이 위험에 처할 수 있습니다. 반면 롤업은 이더리움 L1의 보안을 상속받습니다. 대규모 자금을 이동할 때는 이 보안 차이를 반드시 고려하세요.
:::

## 상태 채널 (State Channels)

두 당사자 간 오프체인에서 다수의 트랜잭션을 처리한 후, 최종 결과만 온체인에 기록하는 방식입니다.

**비트코인 라이트닝 네트워크**: 가장 대표적인 상태 채널 구현체로, 비트코인의 소액 결제를 즉시 처리합니다. 채널 개설 시 온체인 거래가 필요하지만, 이후 채널 내 거래는 수수료가 거의 없고 즉시 처리됩니다.

**장점**: 즉시 완결성, 극소 수수료, 높은 프라이버시
**단점**: 채널 개설/해제 시 온체인 거래 필요, 상시 온라인 유지 필요, 범용 스마트 컨트랙트 지원 어려움

:::

## 브릿지 리스크와 크로스체인 상호운용성

### 브릿지의 종류

**신뢰 기반 브릿지 (Trusted)**:
- 중앙화된 검증자 세트가 자산 이동을 관리
- 예: Multichain(구 Anyswap), Wormhole
- 속도가 빠르지만, 검증자 해킹 시 대규모 손실 가능

**신뢰 최소화 브릿지 (Trust-Minimized)**:
- L1의 보안에 의존하는 네이티브 롤업 브릿지
- 예: Arbitrum Bridge, Optimism Gateway
- 보안이 강하지만 출금에 7일 대기 (Optimistic Rollup) 또는 증명 생성 시간(ZK Rollup) 필요

**유동성 네트워크 브릿지**:
- 각 체인에 유동성 풀을 유지하여 즉시 교환
- 예: Across, Stargate, Hop Protocol
- 빠른 속도와 합리적 수수료, 유동성 한도 존재

### 브릿지 해킹 사례

| 사건 | 시기 | 피해액 | 원인 |
|-----|------|-------|------|
| Ronin Bridge | 2022.3 | $625M | 검증자 개인키 탈취 |
| Wormhole | 2022.2 | $320M | 스마트 컨트랙트 취약점 |
| Nomad | 2022.8 | $190M | 검증 로직 결함 |
| Harmony Horizon | 2022.6 | $100M | 멀티시그 키 탈취 |

:::danger 브릿지는 가장 큰 보안 취약점
크로스체인 브릿지는 DeFi에서 가장 많은 해킹 피해가 발생하는 영역입니다. 2022년에만 브릿지 해킹으로 약 20억 달러의 손실이 발생했습니다. 대규모 자산 이동 시 네이티브 브릿지를 사용하고, 제3자 브릿지 사용 시 한 번에 이동하는 금액을 제한하세요.
:::

### 크로스체인 상호운용성

**LayerZero**: 옴니체인(Omnichain) 메시징 프로토콜로, 체인 간 메시지를 전달하여 크로스체인 dApp을 구현합니다. OFT(Omnichain Fungible Token) 표준으로 멀티체인 토큰을 지원합니다.

**Chainlink CCIP**: Chainlink의 크로스체인 상호운용성 프로토콜로, 토큰 전송과 임의의 메시지를 체인 간에 전달합니다. Chainlink의 오라클 네트워크를 활용한 보안이 특징입니다.

**IBC (Inter-Blockchain Communication)**: Cosmos 생태계의 크로스체인 통신 프로토콜로, Cosmos SDK 기반 체인 간 자산과 메시지 전송을 지원합니다.

## L2 생태계 투자 분석

### L2 토큰 밸류에이션 프레임워크

L2 토큰의 가치는 다음 요소에 의해 결정됩니다.

1. **수수료 수익** — L2가 사용자에게 받는 가스비와 L1에 지불하는 데이터 게시 비용의 차이 (순수익)
2. **MEV 수익** — L2 시퀀서가 포착하는 MEV 수익
3. **생태계 규모** — TVL, 활성 사용자 수, 거래 건수
4. **토큰 유틸리티** — 거버넌스, 스테이킹, 수수료 지불 등
5. **네트워크 효과** — 개발자 수, dApp 수, 고유 사용자 수

**주요 L2 수익 비교 (연간화 기준)**:

```
수수료 순수익 = 사용자 가스비 수입 - L1 데이터 게시 비용

Arbitrum: 연간 수수료 수입 ~$50M+, 데이터 비용 차감 후 순수익 ~$30M+
Optimism: 연간 수수료 수입 ~$30M+, 순수익 ~$20M+
Base: 연간 수수료 수입 ~$40M+, Coinbase로 수익 귀속
```

### Superchain과 L2 생태계 경쟁

**OP Superchain**: Optimism의 OP Stack을 사용하는 L2들의 연합 네트워크입니다. Base, Zora, Mode 등이 참여하며, 상호운용성과 공유 보안을 목표로 합니다.

**Arbitrum Orbit**: Arbitrum의 기술을 활용한 L3(레이어3) 또는 커스텀 체인 구축 프레임워크입니다.

**zkSync Hyperchains**: zkSync의 기술을 활용한 커스텀 ZK 체인 생태계입니다.

## L2 선택 가이드

### 사용 목적별 추천

- **일반 DeFi 사용** — Arbitrum (최대 생태계), Base (낮은 수수료)
- **NFT/게임** — Immutable X (ZK 기반 NFT 특화), Base
- **프라이버시** — zkSync, Aztec (ZK 프라이버시 특화)
- **기관 용도** — Polygon zkEVM (규제 준수 친화적), Base
- **개발자** — Arbitrum (가장 넓은 도구 지원), StarkNet (카이로 혁신)

### 주요 고려 사항

1. **보안 성숙도** — 롤업의 탈중앙화 수준 (시퀀서 단일 장애점, 탈출 해치 유무)
2. **생태계 규모** — 사용 가능한 dApp과 유동성
3. **가스비** — 사용 빈도에 따른 비용 부담
4. **출금 시간** — 자금의 유동성 요구 사항
5. **개발자 지원** — 문서화, 도구, 커뮤니티 활성도
