# 암호화폐 보안 & 지갑 관리

"Not your keys, not your coins" — 암호화폐의 보안은 전통 금융과 근본적으로 다릅니다. 은행이 자산을 보호해주는 전통 금융과 달리, 암호화폐에서는 개인이 자신의 자산을 직접 관리하고 보호해야 합니다. 해킹, 피싱, 러그풀 등으로 인한 손실은 되돌릴 수 없는 경우가 대부분입니다. 이 가이드에서는 암호화폐 자산을 안전하게 관리하기 위한 종합적인 보안 전략을 다룹니다.

---

## 지갑의 종류와 특성

### 핫 월렛 (Hot Wallet)

인터넷에 연결된 지갑으로, 편리하지만 해킹 위험에 노출됩니다.

**소프트웨어 지갑**:

| 지갑 | 유형 | 지원 체인 | 특징 |
|-----|------|---------|------|
| MetaMask | 브라우저/모바일 | EVM 체인 | 가장 널리 사용, DeFi 필수 |
| Phantom | 브라우저/모바일 | Solana, Ethereum | 솔라나 생태계 주력 |
| Rabby | 브라우저 | EVM 체인 | 트랜잭션 시뮬레이션, 보안 경고 |
| Trust Wallet | 모바일 | 멀티체인 | 바이낸스 계열, 직관적 UI |
| Keplr | 브라우저/모바일 | Cosmos 체인 | Cosmos 생태계 표준 |

**핫 월렛 보안 수칙**:
1. 공식 사이트/앱스토어에서만 다운로드
2. 시드 구문을 절대 디지털 형태로 저장하지 않음
3. 사용하지 않는 토큰 승인(Approval) 정기적으로 해제 (Revoke.cash 활용)
4. DeFi 상호작용 전 트랜잭션 내용 확인
5. 대규모 자산은 핫 월렛에 보관하지 않음

### 콜드 월렛 (Cold Wallet)

인터넷에 연결되지 않는 지갑으로, 가장 높은 수준의 보안을 제공합니다.

**하드웨어 지갑**:

| 제품 | 가격대 | 보안 칩 | 특징 |
|-----|-------|--------|------|
| Ledger Nano X | ~$149 | CC EAL5+ (Secure Element) | 블루투스, 모바일 지원 |
| Ledger Nano S Plus | ~$79 | CC EAL5+ | 유선, 가성비 |
| Ledger Stax | ~$279 | CC EAL5+ | E-Ink 터치스크린 |
| Trezor Model T | ~$169 | 일반 마이크로컨트롤러 | 오픈소스 펌웨어, 터치스크린 |
| Trezor Safe 3 | ~$79 | Secure Element 추가 | Trezor 최신 모델 |
| Keystone Pro | ~$169 | CC EAL5+ | 에어갭(QR코드), 오픈소스 |

:::tip 하드웨어 지갑 선택 기준
**Ledger**: Secure Element 칩을 사용하여 물리적 공격에 강하지만, 펌웨어가 오픈소스가 아닌 점이 비판의 대상입니다. 2023년 Ledger Connect Kit 공급망 해킹 사건 이후 보안 검증 절차가 강화되었습니다.

**Trezor**: 펌웨어가 오픈소스로 커뮤니티 검증이 가능하지만, 물리적 접근 시 취약점이 보고된 적 있습니다. Safe 3부터 Secure Element을 추가했습니다.

**에어갭 지갑(Keystone 등)**: USB나 블루투스 없이 QR코드로만 통신하여 물리적 연결을 완전히 차단합니다.
:::

**페이퍼 월렛**: 개인키나 시드 구문을 종이에 기록하여 오프라인 보관하는 가장 원시적인 형태입니다. 화재, 수해, 물리적 훼손에 취약하므로 금속 백업(스틸 플레이트)이 권장됩니다.

## 멀티시그 (Multi-Signature)

### 개념

멀티시그는 트랜잭션 실행에 여러 개의 서명(키)이 필요한 보안 메커니즘입니다. "M-of-N" 구조로, N개의 키 중 M개의 서명이 있어야 거래가 승인됩니다.

**일반적인 구성**:
- **2-of-3** — 3개 키 중 2개 서명 필요 (개인 사용 권장)
- **3-of-5** — 5개 키 중 3개 서명 필요 (조직/DAO 사용)
- **4-of-7** — 7개 키 중 4개 서명 필요 (대규모 프로토콜 금고)

**주요 멀티시그 솔루션**:

| 솔루션 | 체인 | 특징 |
|-------|------|------|
| Safe (구 Gnosis Safe) | EVM 체인 | 업계 표준, 가장 많은 TVL 보호 |
| Squads | Solana | 솔라나 멀티시그 표준 |
| Bitcoin 네이티브 | Bitcoin | PSBT 기반 네이티브 멀티시그 |

**멀티시그 설정 전략**:
```
개인 자산 보호 (2-of-3):
- 키 1: 하드웨어 지갑 A (Ledger) — 자택 보관
- 키 2: 하드웨어 지갑 B (Trezor) — 별도 장소 보관
- 키 3: 시드 구문 금속 백업 — 은행 금고

→ 하나의 기기를 분실해도 나머지 2개로 자산 접근 가능
→ 해커가 하나의 키를 탈취해도 자산 안전
```

### MPC 지갑 (Multi-Party Computation)

MPC는 개인키를 여러 조각(Key Share)으로 나누어 각 당사자가 보유하고, 서명 시 개인키를 재구성하지 않고도 각자의 조각으로 부분 서명을 생성하여 합산하는 기술입니다.

**멀티시그 vs MPC 비교**:

| 항목 | 멀티시그 | MPC |
|-----|---------|-----|
| 체인 호환성 | 체인별 구현 필요 | 체인 무관 (범용) |
| 온체인 노출 | 멀티시그 구조 공개 | 일반 지갑처럼 보임 |
| 가스비 | 서명 수만큼 증가 | 단일 서명과 동일 |
| 키 교체 | 새 지갑 생성 필요 | 키 조각만 교체 가능 |
| 성숙도 | 높음 (검증됨) | 중간 (발전 중) |

**주요 MPC 지갑**: Fireblocks (기관용), Zengo (개인용), Fordefi (기관용)

## 시드 구문 (Seed Phrase) 관리

### 시드 구문의 중요성

시드 구문(Recovery Phrase, Mnemonic)은 일반적으로 12개 또는 24개의 영단어로 구성되며, 지갑의 모든 개인키를 복원할 수 있는 마스터 키입니다.

:::danger 시드 구문 관리의 절대 원칙
1. **절대 디지털 형태로 저장하지 마세요** — 스크린샷, 메모 앱, 클라우드, 이메일 금지
2. **절대 누구에게도 공유하지 마세요** — 어떤 서비스도 시드 구문을 요구하지 않습니다
3. **절대 웹사이트에 입력하지 마세요** — 피싱 사이트의 가장 흔한 수법입니다
4. **물리적으로 안전하게 보관하세요** — 금속 백업, 복수 장소 분산 보관
5. **상속 계획을 세우세요** — 불의의 사고 시 가족이 자산에 접근할 수 있도록
:::

### 백업 전략

**기본 백업 (개인)**:
- 종이에 시드 구문 기록 → 방수 봉투 → 금고 보관
- 금속 백업 플레이트(CryptoSteel, Billfodl 등) → 화재/수해 방지

**고급 백업 (샤미르 시크릿 쉐어링)**:
- 시드 구문을 여러 조각으로 분할 (예: 3개 중 2개로 복원)
- 각 조각을 다른 장소/사람에게 분산 보관
- Trezor의 Shamir Backup이 대표적 구현

**상속 계획**:
- 변호사에게 지갑 접근 방법을 밀봉 문서로 위탁
- 가족에게 하드웨어 지갑 위치와 PIN을 분리하여 전달
- 타임록 컨트랙트를 활용한 자동 상속 (Dead Man's Switch)

## 주요 공격 벡터와 방어

### 피싱 (Phishing)

암호화폐 사용자를 대상으로 한 가장 흔하고 효과적인 공격입니다.

**유형**:
- **가짜 웹사이트** — 공식 사이트와 유사한 URL(예: uniswаp.org - 키릴 문자 'а' 사용)
- **가짜 에어드랍** — 지갑에 무작위로 토큰을 보내고, 클레임 시 악성 트랜잭션 유도
- **소셜 미디어** — 공식 계정을 사칭한 DM, 가짜 고객센터
- **이메일** — 거래소/지갑 서비스를 사칭한 피싱 메일

**방어법**:
1. URL을 항상 직접 입력하거나 북마크 사용
2. 지갑 연결 요청을 받으면 반드시 URL 확인
3. 알 수 없는 토큰에 대한 상호작용 금지
4. 하드웨어 지갑 화면에서 트랜잭션 내용 직접 확인
5. Pocket Universe, Wallet Guard 같은 트랜잭션 시뮬레이션 도구 활용

### 러그풀 (Rug Pull)

프로젝트 개발자가 투자금을 모은 후 잠적하거나, 유동성을 빼돌리는 사기입니다.

**유형**:
- **유동성 인출** — DEX 풀에서 유동성을 제거하여 토큰을 매도 불가하게 만듦
- **매도 불가 토큰** — 스마트 컨트랙트에 매수만 가능하고 매도를 차단하는 코드 삽입
- **민팅 백도어** — 개발자만 무제한 토큰을 발행할 수 있는 함수
- **숨겨진 소유자 권한** — 프록시 패턴을 악용한 컨트랙트 변경

**러그풀 감별법**:
1. 컨트랙트 소스코드 검증 여부 확인 (Etherscan Verified)
2. 유동성 락 여부 (Team.finance, Unicrypt 등으로 확인)
3. 소유자 권한 존재 여부 (mint, pause, blacklist 함수)
4. 팀 신원 확인 (익명 팀은 리스크 높음)
5. TokenSniffer, GoPlus Security 같은 자동 탐지 도구 활용

### 프론트러닝 (Front-running) & MEV

**프론트러닝**: 사용자의 대기 중인 트랜잭션을 미리 확인하고, 더 높은 가스비를 지불하여 앞서 실행하는 공격입니다.

**샌드위치 공격**: 타겟 트랜잭션의 앞뒤에 공격자의 트랜잭션을 끼워넣어 가격을 조작하고 차익을 얻습니다.

```
공격 과정:
1. 사용자: ETH → USDC 스왑 트랜잭션 제출 (멤풀에 대기)
2. 공격자: 사용자 앞에 ETH → USDC 스왑 실행 (가격 상승)
3. 사용자: 더 높은 가격에 스왑 실행 (슬리피지 발생)
4. 공격자: 사용자 뒤에 USDC → ETH 스왑 실행 (차익 실현)
```

**방어법**:
- 슬리피지를 최소로 설정 (0.5~1%)
- MEV 보호 RPC 사용 (Flashbots Protect, MEV Blocker)
- Private 트랜잭션 서비스 활용
- 대규모 스왑은 분할 실행

### 스마트 컨트랙트 취약점

| 취약점 | 설명 | 대표 사례 |
|-------|------|---------|
| 재진입 (Reentrancy) | 외부 호출 중 상태 변경 전에 재호출 | The DAO 해킹 ($60M) |
| 오라클 조작 | 가격 오라클을 조작하여 비정상 거래 | Mango Markets ($114M) |
| 플래시론 공격 | 대규모 무담보 대출로 프로토콜 조작 | Pancake Bunny ($45M) |
| 로직 오류 | 비즈니스 로직의 버그 | Wormhole ($320M) |
| 접근 제어 | 관리자 함수의 부적절한 접근 제어 | Ronin Bridge ($625M) |

:::caution 스마트 컨트랙트와 상호작용할 때
새로운 프로토콜이나 컨트랙트와 상호작용하기 전에 반드시 다음을 확인하세요.

1. 컨트랙트 소스코드가 공개되어 있는지
2. 신뢰할 수 있는 기관의 감사를 받았는지
3. 충분한 기간 동안 운영되어 실전 검증(Battle-tested)되었는지
4. 토큰 승인(Approve) 시 무제한(Unlimited) 대신 필요한 양만 승인
5. 사용 후 불필요한 승인은 Revoke
:::

## 거래소 보안 평가

### 거래소 선택 시 확인 사항

1. **자산 관리** — 콜드 월렛 보관 비율 (90% 이상 권장), Proof of Reserves
2. **보안 인프라** — 2FA 지원, 출금 화이트리스트, 출금 대기 시간, 이상 거래 탐지
3. **보험/보상** — 해킹 시 보상 정책, 보험 가입 여부
4. **규제 준수** — 라이선스 보유, AML/KYC 절차
5. **보안 이력** — 과거 해킹 사건과 대응 방식
6. **투명성** — PoR(Proof of Reserves) 공개, 감사 보고서

### 한국 거래소 규제 요건

**가상자산이용자보호법 (2024년 시행)** 주요 요건:

- **이용자 예치금 분리보관** — 이용자 자산을 거래소 자산과 분리
- **콜드 월렛 보관** — 이용자 가상자산의 80% 이상 콜드 월렛 보관
- **보험/공제 가입** — 해킹 등 사고 시 보상을 위한 보험 의무 가입
- **이상거래 감시** — 시세조종, 내부거래 등 불공정거래 모니터링
- **정보보호 관리체계(ISMS)** — 인증 취득 의무

## 보험 프로토콜

DeFi 보안 리스크를 헤지하기 위한 탈중앙화 보험 프로토콜이 있습니다.

| 프로토콜 | 보장 범위 | 특징 |
|---------|---------|------|
| Nexus Mutual | 스마트컨트랙트 해킹, 오라클 장애 | 최대 규모, 상호보험 모델 |
| InsurAce | 스마트컨트랙트, 거래소, 스테이블 디페깅 | 크로스체인 보장 |
| Unslashed Finance | 거래소 해킹, 오라클 장애 | 자본 효율적 모델 |

:::tip 보험 비용 대비 가치
DeFi 보험의 연간 보험료는 보장 금액의 2~10% 수준입니다. 대규모 자금을 DeFi에 예치하는 경우 보험 가입을 고려할 가치가 있습니다. 다만 보험 프로토콜 자체의 스마트 컨트랙트 리스크도 존재하므로, 보험이 절대적 안전을 보장하지는 않습니다.
:::

## 보안 체크리스트

### 초보자용 (최소 보안)

- [ ] 거래소 2FA 설정 (Google Authenticator, SMS가 아닌 앱 기반)
- [ ] 강력하고 고유한 비밀번호 (패스워드 매니저 사용)
- [ ] 공식 앱/사이트만 사용 (북마크 활용)
- [ ] 시드 구문 종이 백업 및 안전 보관
- [ ] 피싱 메일/DM 주의

### 중급자용 (권장 보안)

- [ ] 하드웨어 지갑 사용 (장기 보유분)
- [ ] 거래소 출금 화이트리스트 설정
- [ ] 토큰 승인 정기 점검 (Revoke.cash)
- [ ] MEV 보호 RPC 사용
- [ ] 지갑 분리 (DeFi용, 보관용, NFT 민팅용)

### 고급자용 (최대 보안)

- [ ] 멀티시그 또는 MPC 지갑
- [ ] 에어갭 하드웨어 지갑
- [ ] 샤미르 시크릿 쉐어링 백업
- [ ] 전용 기기에서만 암호화폐 관리
- [ ] DeFi 보험 가입
- [ ] 상속 계획 수립
