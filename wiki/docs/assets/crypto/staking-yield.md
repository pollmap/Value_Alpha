# 스테이킹 & 수익 전략

스테이킹(Staking)은 지분증명(Proof of Stake, PoS) 블록체인에서 네트워크 보안에 기여하고 보상을 받는 행위입니다. 이더리움이 2022년 The Merge를 통해 PoS로 전환한 이후, 스테이킹은 암호화폐 투자에서 가장 핵심적인 수익 전략 중 하나가 되었습니다. 이 가이드에서는 스테이킹의 기술적 메커니즘부터 고급 수익 전략, 리스크 관리까지 체계적으로 다룹니다.

---

## PoS 스테이킹 메커니즘

### 지분증명(PoS)의 원리

PoS에서 검증자(Validator)는 자신의 자산을 담보로 잠그고(스테이킹), 블록 생성 및 트랜잭션 검증에 참여합니다. 정직한 행동에 대해 보상을 받고, 악의적 행동(이중 서명, 다운타임 등)에 대해 담보가 삭감(슬래싱)됩니다.

**이더리움 PoS 핵심 사양**:

- **최소 스테이킹량** — 32 ETH (검증자 1노드 기준)
- **연간 보상률** — 약 3~5% (총 스테이킹량에 따라 변동)
- **검증자 수** — 약 100만 개 이상 (2024년 기준)
- **총 스테이킹 ETH** — 약 3,300만 ETH (전체 공급의 약 28%)
- **출금 대기** — 활성화 큐와 퇴장 큐에 따라 수일~수주
- **슬래싱 패널티** — 최소 1/32 ETH, 동시 슬래싱 수에 따라 최대 전액

### 검증자 경제학

검증자 수익은 다음 요소로 구성됩니다.

**합의 보상 (Consensus Rewards)**:
- 블록 제안(Block Proposal) 보상
- 증명(Attestation) 보상
- 동기화 위원회(Sync Committee) 보상

**실행 보상 (Execution Rewards)**:
- 우선순위 수수료(Priority Fee / Tips)
- MEV 수익 (MEV-Boost를 통해)

```
검증자 연간 수익 = 합의 보상 + 실행 보상

합의 보상 ≈ 기본 보상률 × 32 ETH
실행 보상 = 블록 제안 시 우선순위 수수료 + MEV

실제 APR ≈ 3~5% (합의) + 1~2% (실행, 변동 큼) = 4~7%
```

:::tip MEV-Boost의 중요성
MEV-Boost를 사용하는 검증자는 사용하지 않는 검증자 대비 평균 20~40% 높은 수익을 얻습니다. MEV-Boost는 블록 빌더들이 경쟁적으로 블록을 구성하고, 가장 높은 가치의 블록을 검증자에게 제안하는 시스템입니다. 대부분의 이더리움 검증자(90% 이상)가 MEV-Boost를 사용하고 있습니다.
:::

### 슬래싱 리스크

슬래싱(Slashing)은 검증자의 악의적 또는 과실 행동에 대한 처벌 메커니즘입니다.

**슬래싱 발동 조건**:
1. **이중 제안(Double Proposal)** — 같은 슬롯에 두 개의 블록을 제안
2. **이중 투표(Double Vote)** — 같은 에폭에서 상충하는 두 증명에 서명
3. **서라운드 투표(Surround Vote)** — 다른 증명을 둘러싸는 증명에 서명

**슬래싱 패널티 구조**:
- 초기 패널티: 스테이킹 잔액의 1/32
- 상관관계 패널티: 동시에 슬래싱되는 검증자 수에 비례 (최대 전액 몰수)
- 증명 미참여 패널티: 슬래싱 후 36일간 보상 제외

:::danger 운영 리스크 관리
개인 검증자 운영 시 이중 서명을 방지하기 위해 **절대로** 동일한 검증자 키를 두 개 이상의 노드에서 동시에 사용하지 마세요. 키 관리, 장애 복구, 업데이트 절차에 대한 명확한 운영 프로토콜을 수립하는 것이 필수적입니다.
:::

## 리퀴드 스테이킹 (Liquid Staking)

### 개념과 필요성

전통적 스테이킹은 자산이 잠겨(locked) DeFi 등 다른 용도로 활용할 수 없습니다. 리퀴드 스테이킹은 스테이킹한 자산의 대표 토큰(LST, Liquid Staking Token)을 발행하여 유동성을 유지하면서 스테이킹 보상을 받을 수 있게 합니다.

```
ETH → Lido 스테이킹 → stETH 수령 (1:1 비율)
                         ↓
              stETH로 DeFi 활동 가능
              (대출 담보, 유동성 공급, 거래 등)
              + 스테이킹 보상 자동 누적
```

### 주요 리퀴드 스테이킹 프로토콜

**Lido (stETH)**

- 시장 점유율: 전체 이더리움 스테이킹의 약 28~30%
- 리베이싱(Rebasing) 모델: stETH 수량이 매일 증가하여 보상 반영
- wstETH: 래핑된 stETH로, DeFi 호환성이 더 높음 (가치 증가 모델)
- 수수료: 스테이킹 보상의 10% (노드 운영자 5% + Lido DAO 5%)
- 탈중앙화 논란: 시장 점유율이 높아 이더리움 검열 저항성에 대한 우려

**Rocket Pool (rETH)**

- 허가 없는(Permissionless) 노드 운영자 참여 가능
- 최소 8 ETH + RPL 보증금으로 미니풀(Minipool) 운영
- rETH: 가치 증가(Value-accruing) 모델로, 토큰 수량은 동일하지만 가치가 상승
- 더 높은 탈중앙화 수준이지만, 수수료가 약간 높음 (14~15%)

**기타 프로토콜**:

| 프로토콜 | 토큰 | 특징 |
|---------|------|------|
| Coinbase (cbETH) | cbETH | 기관 친화적, 미국 규제 준수 |
| Frax Ether | sfrxETH | Frax 생태계 통합, 듀얼 토큰 모델 |
| Mantle (mETH) | mETH | Mantle L2 생태계 통합 |
| Swell | swETH | 리스테이킹 통합, 에어드랍 포인트 |

### LST의 리스크

1. **디페깅 리스크** — stETH 등 LST가 ETH과의 1:1 가치를 잃을 수 있음 (2022년 stETH 디스카운트 최대 6%)
2. **스마트 컨트랙트 리스크** — 프로토콜 취약점으로 인한 자금 손실
3. **중앙화 리스크** — 특정 프로토콜에 스테이킹이 집중되면 네트워크 검열 위험
4. **규제 리스크** — SEC 등 규제기관이 LST를 증권으로 분류할 가능성
5. **슬래싱 전가** — 노드 운영자의 슬래싱이 LST 보유자에게 전가될 수 있음

## 리스테이킹 (Restaking)

### EigenLayer와 리스테이킹 개념

리스테이킹은 이미 스테이킹된 ETH(또는 LST)의 보안을 다른 프로토콜에도 제공하여 추가 수익을 얻는 혁신적 메커니즘입니다. EigenLayer가 이 개념을 개척했습니다.

**작동 원리**:

```
ETH 스테이킹 → 이더리움 보안 제공 → 기본 스테이킹 보상
     +
리스테이킹(EigenLayer) → AVS 보안 제공 → 추가 보상

AVS (Actively Validated Service):
- 오라클 네트워크
- 데이터 가용성 레이어
- 브릿지 검증
- 롤업 시퀀서
- 기타 미들웨어
```

**EigenLayer 참여 방법**:
1. **네이티브 리스테이킹** — 이더리움 검증자가 직접 EigenLayer에 참여
2. **LST 리스테이킹** — stETH, rETH 등 LST를 EigenLayer에 예치
3. **리퀴드 리스테이킹 토큰(LRT)** — EtherFi(eETH), Puffer(pufETH) 등을 통해 리스테이킹 + 유동성 유지

### 리스테이킹의 리스크

:::caution 겹겹이 쌓이는 리스크
리스테이킹은 수익을 극대화하지만, 리스크도 중첩됩니다.

1. **중첩 슬래싱** — 이더리움 슬래싱 + AVS 슬래싱이 동시에 발생할 수 있음
2. **시스템 리스크** — 대규모 슬래싱 이벤트가 연쇄 청산을 유발할 가능성
3. **복잡성 리스크** — ETH → LST → LRT → DeFi의 각 단계에서 스마트 컨트랙트 리스크 누적
4. **상관관계 리스크** — 여러 AVS에서 동시에 문제가 발생할 수 있음
5. **유동성 리스크** — LRT의 유동성이 부족하면 출구 전략이 제한됨

리스테이킹에 투자하기 전 각 레이어의 리스크를 개별적으로 평가하고, 전체 포지션 크기를 보수적으로 관리하세요.
:::

## 수익 원천 분류 (Yield Taxonomy)

### 실질 수익 (Real Yield) vs 인플레이션 수익

DeFi 수익을 올바르게 평가하려면 수익의 원천을 정확히 이해해야 합니다.

**실질 수익 (Real Yield)**:
- 거래 수수료 (DEX 유동성 공급)
- 대출 이자 (렌딩 프로토콜)
- 프로토콜 수익 분배
- 청산 수익
- MEV 수익

**인플레이션 수익 (Inflationary Yield)**:
- 신규 토큰 발행 보상 (유동성 마이닝)
- PoS 스테이킹 보상 (네트워크 인플레이션)
- 에어드랍 기대 (포인트 파밍)

```
실질 수익률 = 명목 APY - 토큰 인플레이션율

예: 스테이킹 APY 5%, 토큰 연간 인플레이션 3%
→ 실질 수익률 = 5% - 3% = 2%
```

### 리스크 조정 수익 비교

| 전략 | 예상 APY | 주요 리스크 | 리스크 수준 |
|------|---------|-----------|-----------|
| ETH 네이티브 스테이킹 | 3~5% | 슬래싱, 운영 | 낮음 |
| 리퀴드 스테이킹 (Lido) | 3~5% | 스마트컨트랙트, 디페깅 | 낮음~중간 |
| 스테이블 렌딩 (Aave) | 2~8% | 스마트컨트랙트, 프로토콜 | 중간 |
| LP (주요 페어) | 5~20% | IL, 스마트컨트랙트 | 중간~높음 |
| 리스테이킹 (EigenLayer) | 5~10% | 중첩 슬래싱, 복잡성 | 중간~높음 |
| LP (마이너 페어) | 20~100%+ | IL, 러그풀, 토큰 하락 | 높음 |
| 레버리지 파밍 | 30~200%+ | 청산, IL, 복합 리스크 | 매우 높음 |

## APY 계산 및 비교 방법론

### 복합 수익률 계산

여러 전략을 결합했을 때의 총 수익률 계산입니다.

```
예시: ETH → stETH → Aave 담보 → USDC 빌리기 → Curve LP

1단계: stETH 스테이킹 수익 ≈ 3.5% APY
2단계: Aave 담보 대출 비용 ≈ -2% (USDC 빌리기)
3단계: Curve LP 수익 ≈ 5% APY (USDC 기반 풀)
4단계: CRV 보상 ≈ 3% APY

총 APY ≈ 3.5% + (LTV × (5% + 3% - 2%))
LTV 50% 기준 ≈ 3.5% + 0.5 × 6% = 6.5%
```

:::tip 수익률 추적 도구
실시간 DeFi 수익률을 비교하려면 다음 도구들을 활용하세요.

- **DefiLlama Yields** — 체인별, 프로토콜별 수익률 비교
- **Dune Analytics** — 온체인 데이터 기반 커스텀 대시보드
- **Token Terminal** — 프로토콜 재무 데이터 및 밸류에이션
- **Rated.network** — 이더리움 검증자별 성과 비교
:::

## 스테이킹 전략 가이드

### 보수적 전략 (리스크 최소화)

1. 이더리움 네이티브 스테이킹 또는 Lido/Rocket Pool LST
2. 스테이블코인 렌딩 (Aave, Compound에 USDC/USDT 예치)
3. 높은 상관관계 자산 쌍 LP (stETH/ETH, USDC/USDT)

### 균형 전략 (리스크-수익 균형)

1. LST + DeFi 결합 (stETH 담보 대출 + 스테이블 LP)
2. 주요 자산 쌍 LP (ETH/USDC Uniswap V3 범위 관리)
3. 리스테이킹 일부 참여 (전체 포지션의 20~30%)

### 공격적 전략 (수익 극대화)

1. 레버리지 리퀴드 스테이킹 (루핑)
2. 신규 프로토콜 유동성 마이닝 (에어드랍 파밍)
3. LRT + DeFi 복합 전략
4. 마이너 체인 스테이킹 (높은 APY, 높은 리스크)

:::danger 레버리지 루핑의 위험
stETH를 Aave에 담보로 넣고 ETH를 빌려 다시 stETH를 만드는 "루핑" 전략은 수익률을 크게 높일 수 있지만, stETH 디페깅 시 연쇄 청산의 위험이 있습니다. 2022년 3AC 사태 당시 이러한 전략으로 대규모 손실이 발생했습니다.
:::

## 한국 투자자를 위한 고려사항

### 세금 관련

- 스테이킹 보상의 과세 분류(기타소득 vs 양도소득)가 아직 불명확
- 해외 리퀴드 스테이킹 프로토콜 사용 시 해외 금융계좌 신고 의무 검토 필요
- 보상 수령 시점 vs 매도 시점의 과세 기준 확인 필요

### 접근성

- 국내 거래소(업비트, 빗썸)에서는 일부 PoS 코인의 스테이킹 서비스 제공
- 이더리움 리퀴드 스테이킹은 주로 해외 프로토콜을 통해 접근
- 하드웨어 지갑 + MetaMask 조합으로 DeFi 직접 참여 가능
