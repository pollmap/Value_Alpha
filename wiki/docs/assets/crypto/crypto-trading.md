# 암호화폐 트레이딩 전략

암호화폐 시장은 24시간 365일 운영되며, 전통 금융 시장 대비 높은 변동성을 보입니다. 이러한 특성은 트레이더에게 기회이자 위험입니다. 이 가이드에서는 암호화폐 트레이딩의 기초부터 고급 전략까지, 한국 시장의 특수성을 포함하여 체계적으로 다룹니다.

## CEX vs DEX 트레이딩

### 중앙화 거래소 (CEX)

중앙화 거래소는 전통적인 주문서(Order Book) 방식으로 운영되며, 높은 유동성과 빠른 체결 속도가 특징입니다.

**글로벌 주요 CEX**:

| 거래소 | 특징 | 현물 거래량 | 파생상품 |
|-------|------|-----------|---------|
| Binance | 최대 거래소, 다양한 상품 | $15B+/일 | 선물, 옵션 |
| Coinbase | 미국 상장, 규제 준수 | $2B+/일 | 제한적 |
| OKX | 파생상품 강점 | $3B+/일 | 선물, 옵션, 스왑 |
| Bybit | 파생상품 특화 | $5B+/일 | 선물, 옵션 |
| Kraken | 유럽 강세, 보안 우수 | $1B+/일 | 선물 |

**CEX의 장점**: 높은 유동성, 빠른 체결, 다양한 주문 유형, 파생상품 지원, 법정화폐 온/오프램프
**CEX의 단점**: 자산 관리권 위임(Not your keys), 해킹 리스크, KYC 요구, 거래 중단 가능성

### 탈중앙화 거래소 (DEX)

DEX는 스마트 컨트랙트를 통해 사용자 간 직접 거래를 중개합니다.

**현물 DEX**: Uniswap, Curve, PancakeSwap, Raydium
**파생상품 DEX**: dYdX, GMX, Hyperliquid, Jupiter Perps

**DEX의 장점**: 자기 수탁(Self-custody), 무허가 상장, 프라이버시, 글로벌 접근
**DEX의 단점**: 높은 가스비(L1), 슬리피지, MEV 공격, 제한된 주문 유형

:::tip CEX와 DEX의 하이브리드 접근
대부분의 경험 있는 트레이더는 CEX와 DEX를 병행합니다. 고빈도 트레이딩이나 파생상품은 CEX에서, 신규 토큰 매수나 에어드랍 파밍은 DEX에서 실행합니다. 큰 금액은 DEX나 하드웨어 지갑에 보관하되, 트레이딩에 필요한 자금만 CEX에 배치하는 것이 리스크 관리의 기본입니다.
:::

## 주문 유형

### 기본 주문

- **시장가(Market Order)** — 현재 최우선 가격으로 즉시 체결. 유동성이 낮을 때 슬리피지 발생 가능
- **지정가(Limit Order)** — 특정 가격 이상/이하에서만 체결. 체결 보장 없음
- **스탑 주문(Stop Order)** — 특정 가격에 도달하면 시장가/지정가 주문이 발동. 손절에 활용

### 고급 주문

- **스탑 리밋(Stop-Limit)** — 트리거 가격과 체결 가격을 분리. 급격한 슬리피지 방지
- **트레일링 스탑(Trailing Stop)** — 가격이 유리한 방향으로 움직일 때 스탑 가격이 따라감
- **OCO (One-Cancels-Other)** — 이익 실현과 손절 주문을 동시에 설정, 하나가 체결되면 나머지 취소
- **TWAP (Time-Weighted Average Price)** — 대량 주문을 시간에 걸쳐 분산 실행
- **빙산 주문(Iceberg)** — 전체 물량 중 일부만 시장에 노출하여 시장 충격 최소화

## 레버리지 & 마진 트레이딩

### 기본 개념

레버리지(Leverage)는 보유 자금보다 큰 포지션을 개설할 수 있게 하는 메커니즘입니다.

```
레버리지 배율 = 포지션 크기 / 증거금

예: 1,000 USDT 증거금, 10배 레버리지 → 10,000 USDT 포지션

수익/손실 = 가격변동률 × 레버리지 배율
1% 상승 시: +10% 수익 (10배 레버리지)
1% 하락 시: -10% 손실 (10배 레버리지)
```

### 마진 유형

**격리 마진(Isolated Margin)**: 해당 포지션에 할당된 증거금만 손실 가능. 최대 손실이 초기 증거금으로 제한됨.

**교차 마진(Cross Margin)**: 계정 전체 잔액이 증거금으로 사용됨. 청산 가격이 더 멀어지지만, 계정 전체가 위험에 노출.

### 펀딩비 (Funding Rate)

무기한 선물(Perpetual Futures)에서 현물 가격과의 괴리를 조정하는 메커니즘입니다.

```
펀딩비 > 0 (양수): 롱 포지션이 숏 포지션에 지급 → 시장 과열 (롱이 많음)
펀딩비 < 0 (음수): 숏 포지션이 롱 포지션에 지급 → 시장 과매도 (숏이 많음)

펀딩비는 보통 8시간마다 정산
연환산 비용 = 8시간 펀딩비 × 3 × 365
```

:::caution 높은 펀딩비의 의미
펀딩비가 0.1%를 넘어가면 연환산 100% 이상의 비용입니다. 롱 포지션 유지 시 펀딩비 부담이 수익을 잠식할 수 있습니다. 반대로, 숏 포지션은 높은 펀딩비를 수취하여 수익원으로 활용할 수 있습니다 (펀딩비 차익거래 전략).
:::

### 청산 메커니즘

```
청산 가격 (롱) = 진입가격 × (1 - 1/레버리지 + 유지증거금률)
청산 가격 (숏) = 진입가격 × (1 + 1/레버리지 - 유지증거금률)

예: BTC $50,000에 10배 롱, 유지증거금 0.5%
청산 가격 ≈ $50,000 × (1 - 0.1 + 0.005) = $45,250
→ 약 9.5% 하락 시 청산
```

:::danger 레버리지 트레이딩의 위험
암호화폐 시장에서 10% 이상의 일간 변동은 흔합니다. 비트코인도 역사적으로 하루 만에 30% 이상 하락한 적이 있습니다. 고배율 레버리지는 극도로 위험하며, 초보자는 3배 이하의 레버리지만 사용하는 것을 강력히 권장합니다. 반드시 손절(Stop-Loss)을 설정하고, 전체 포트폴리오의 5% 이상을 단일 레버리지 포지션에 투입하지 마세요.
:::

## 차익거래 (Arbitrage) 전략

### CEX-DEX 차익거래

동일한 토큰이 CEX와 DEX에서 다른 가격에 거래될 때의 차익을 포착합니다.

```
수익 = |CEX 가격 - DEX 가격| - 가스비 - 거래 수수료 - 슬리피지

예: ETH가 CEX에서 $3,000, DEX에서 $3,030일 때
수익 = $30 - $5(가스) - $6(수수료) - $3(슬리피지) = $16/ETH
```

**도전 과제**: 가스비 변동, 슬리피지, 실행 속도, MEV 봇과의 경쟁

### 크로스 거래소 차익거래

서로 다른 CEX 간의 가격 차이를 이용합니다.

- 양 거래소에 미리 자금을 배치하는 "사전 배치(Pre-positioning)" 방식이 일반적
- 자금 이체 시간 동안의 가격 변동 리스크를 회피
- 수수료와 출금 수수료를 고려한 순익 계산 필수

### 삼각 차익거래 (Triangular Arbitrage)

세 가지 거래 쌍을 순환하여 차익을 포착합니다.

```
BTC/USDT → ETH/BTC → ETH/USDT

1. 1,000 USDT로 BTC 매수 (BTC/USDT)
2. BTC로 ETH 매수 (ETH/BTC)
3. ETH를 USDT로 매도 (ETH/USDT)
4. 결과: 1,002 USDT (0.2% 수익)
```

실제로는 밀리초 단위의 실행 속도가 필요하며, 봇 프로그래밍이 필수적입니다.

### 김치 프리미엄 (Kimchi Premium) 차익거래

한국 거래소와 글로벌 거래소 간의 가격 차이를 "김치 프리미엄"이라고 합니다.

**김치 프리미엄 계산**:
```
김치 프리미엄(%) = (국내 가격(원화) / (해외 가격(USD) × 환율) - 1) × 100

예: 비트코인 국내 가격 7,000만원, 해외 $50,000, 환율 1,350원
프리미엄 = (70,000,000 / (50,000 × 1,350) - 1) × 100
         = (70,000,000 / 67,500,000 - 1) × 100
         ≈ 3.7%
```

**김치 프리미엄 변동 요인**:
- 국내 투기 심리 과열 시 프리미엄 확대 (2021년 최대 20%+)
- 시장 하락 시 역프리미엄(디스카운트) 발생 가능
- 외국환 규제로 인한 차익거래 제한 (연간 $50,000 해외송금 한도)
- 가상자산 이전 시 트래블룰 적용

:::caution 김치 프리미엄 차익거래의 법적 리스크
김치 프리미엄을 이용한 차익거래는 외국환거래법 위반의 소지가 있습니다. 가상자산을 이용한 외화 환전이나 자금 이체는 불법 환전으로 해석될 수 있으며, 실제로 이와 관련된 기소 사례가 존재합니다. 반드시 법률 전문가의 조언을 구하세요.
:::

## 마켓 메이킹 (Market Making)

### 개념

마켓 메이커는 매수/매도 양쪽에 지정가 주문을 배치하여 유동성을 제공하고, 스프레드(매수-매도 가격 차이)에서 수익을 얻습니다.

**수익 구조**:
```
스프레드 수익 = (매도가 - 매수가) × 거래량
순수익 = 스프레드 수익 - 재고 리스크 비용 - 수수료

예: BTC 매수 $49,990 / 매도 $50,010, 스프레드 $20
일 거래량 100 BTC → 일 스프레드 수익 = $2,000
```

**리스크**: 급격한 가격 변동 시 한쪽 방향으로만 체결되어 재고 리스크(Inventory Risk)가 발생합니다. 재고가 쌓인 방향으로 가격이 불리하게 움직이면 손실이 발생합니다.

### AMM 마켓 메이킹

DEX에서의 유동성 공급은 자동화된 마켓 메이킹입니다. Uniswap V3의 집중 유동성을 활용하면 좁은 가격 범위에서 높은 자본 효율성을 달성할 수 있지만, 가격 범위를 벗어나면 수익이 중단되므로 적극적인 범위 관리가 필요합니다.

## 그리드 트레이딩 (Grid Trading)

### 개념

미리 설정한 가격 범위 내에서 일정 간격으로 매수/매도 주문을 배치하는 자동화 전략입니다. 횡보 시장에서 특히 효과적입니다.

**설정 예시**:
```
범위: $45,000 ~ $55,000
그리드 수: 10
간격: $1,000

$55,000 - 매도 주문
$54,000 - 매도 주문
$53,000 - 매도 주문
...
$47,000 - 매수 주문
$46,000 - 매수 주문
$45,000 - 매수 주문

가격이 $48,000에서 $49,000으로 상승하면:
$48,000 매수 → $49,000 매도 → $1,000 × 수량 수익
```

**장점**: 감정 배제, 횡보장 수익, 자동화
**단점**: 강한 추세장에서 기회 비용, 범위 이탈 리스크, 수수료 부담

## 온체인 분석 기반 트레이딩 시그널

### 주요 온체인 지표

**고래 움직임 모니터링**:
- 대규모 거래소 입금 → 매도 압력 가능성
- 대규모 거래소 출금 → 장기 보유 의향
- 고래 지갑 축적 패턴 분석

**거래소 잔고 변화**:
```
거래소 잔고 증가 → 매도 가능성 증가 (약세 시그널)
거래소 잔고 감소 → 자기 보관 증가 (강세 시그널)
```

**네트워크 활동**:
- 활성 주소 증가 + 가격 상승 → 건전한 상승 추세
- 가격 상승 + 활성 주소 감소 → 소수의 투기적 상승 (약세 다이버전스)

**MVRV (Market Value to Realized Value)**:
- MVRV > 3.0 → 시장 과열 가능성 (차익 실현 고려)
- MVRV < 1.0 → 시장 저평가 가능성 (축적 기회)

## 한국 거래소 트레이딩

### 업비트 (Upbit)

- 한국 최대 거래소 (현물 거래량 기준)
- 두나무(카카오 계열)가 운영
- KRW, BTC, USDT 마켓 제공
- 빠른 신규 코인 상장 → 상장 직후 급등/급락 주의
- API 제한이 엄격하여 고빈도 자동매매에 제한

### 빗썸 (Bithumb)

- 국내 2위 거래소
- 간혹 업비트 대비 가격 차이 발생 (국내 거래소 간 차익거래 기회)
- 다양한 이벤트와 에어드랍

### 한국 거래소 특수성

**실명 확인 계좌**: 국내 거래소는 은행 실명 확인 계좌를 통해서만 원화 입출금이 가능합니다. 각 거래소별 제휴 은행이 다릅니다.

**트래블룰**: 2022년부터 시행된 트래블룰에 따라 100만원 이상의 가상자산 이전 시 송수신자 정보가 전달되어야 합니다. 국내 거래소 간 이전은 비교적 원활하나, 개인 지갑이나 해외 거래소와의 이전은 제한될 수 있습니다.

**거래 시간**: 24/7 운영이지만, 원화 입출금은 은행 영업시간에 따라 제한됩니다.

## DCA (Dollar Cost Averaging) 전략

### 개념

정기적으로 일정 금액을 투자하여 매입 단가를 평균화하는 전략입니다. 시장 타이밍의 어려움을 인정하고 장기적 가치 상승에 베팅하는 접근법입니다.

**효과 분석**:
```
비트코인 매주 10만원 DCA (2019~2024 가정):

총 투자: 10만원 × 52주 × 5년 = 2,600만원
평균 매입가: 가격 변동에 따른 가중 평균
```

역사적으로 비트코인에 대한 4년 이상의 DCA는 항상 양의 수익률을 기록했습니다. 그러나 과거 실적이 미래를 보장하지는 않습니다.

**DCA 최적화**:
- **밸류 DCA** — 가격이 이동평균 아래일 때 더 많이, 위일 때 더 적게 투자
- **멀티 에셋 DCA** — BTC, ETH 등 여러 자산에 분산 DCA
- **자동화** — 거래소의 정기 매수 기능이나 API 봇 활용

## 리스크 관리

### 포지션 사이징

```
최대 손실 허용 = 총 자산 × 리스크 비율 (보통 1~2%)
포지션 크기 = 최대 손실 허용 / (진입가 - 손절가)

예: 총 자산 1억원, 리스크 1%, 손절폭 5%
최대 손실 = 1억 × 0.01 = 100만원
포지션 크기 = 100만원 / 0.05 = 2,000만원
```

### 핵심 리스크 관리 원칙

1. **항상 손절 설정** — 감정이 아닌 사전 계획에 따라 손절
2. **레버리지 제한** — 경험에 관계없이 적절한 레버리지 유지
3. **분산 투자** — 단일 포지션에 전체 자산의 20% 이상 투입 금지
4. **거래소 분산** — 자산을 여러 거래소와 지갑에 분산
5. **기록 관리** — 모든 거래를 기록하고 정기적으로 복기
