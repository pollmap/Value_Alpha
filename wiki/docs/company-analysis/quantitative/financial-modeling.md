# 재무 모델링 기초

## 학습 목표

- 3개년 재무 추정 모델의 기본 구조를 설계할 수 있다
- 매출, 비용, 자본지출의 추정 방법론을 습득한다
- 재무 모델의 정합성을 검증하는 체크리스트를 활용할 수 있다

---


## 재무 모델링이란?

재무 모델링은 기업의 **미래 재무제표를 체계적으로 추정**하는 작업입니다. DCF 밸류에이션의 전단계로, 재무제표 분석(Layer 2)에서 파악한 패턴을 바탕으로 미래를 예측합니다.

## 재무 모델링 원칙

견고한 재무 모델을 구축하기 위해서는 설계 단계부터 명확한 원칙을 수립해야 합니다.

### 핵심 원칙 3가지

| 원칙 | 설명 | 실무 적용 |
|------|------|----------|
| **투명성** | 모든 가정과 산식이 명확하게 드러나야 한다 | 하드코딩 금지, 가정은 별도 셀에 분리 |
| **유연성** | 가정 변경 시 모델 전체가 자동으로 업데이트되어야 한다 | 가정 시트에서 핵심 변수만 변경하면 전체 반영 |
| **감사 가능성** | 제3자가 모델을 검토하고 검증할 수 있어야 한다 | 일관된 서식, 색상 코딩, 주석 활용 |

### 모델 아키텍처 설계

효율적인 모델은 시트 구조부터 체계적으로 설계합니다:

```
[가정 시트] ← 모든 핵심 변수 집중 관리
  ↓
[손익계산서] → [재무상태표] → [현금흐름표]
  ↓                ↓               ↓
[FCF 산출] → [DCF 밸류에이션] → [시나리오 분석]
  ↓
[검증 시트] ← 오류 체크, 밸런스 확인
```

**색상 코딩 규칙**:
- 파란색: 직접 입력 가정 (수동 입력값)
- 검은색: 산식에 의한 계산값
- 녹색: 다른 시트에서 참조하는 링크값
- 빨간색: 오류 체크 또는 경고 플래그

### 시간 축 설계

| 구간 | 기간 | 세분화 |
|------|------|--------|
| 과거 실적 | 최근 3~5년 | 연간 + 최근 4분기 |
| 상세 추정 | 향후 3~5년 | 연간 (필요시 분기) |
| 정상화 구간 | 추정 마지막 1~2년 | 연간 |

모델의 시간 축은 기업 특성에 따라 조정합니다. 반도체처럼 사이클이 뚜렷한 산업은 최소 5년 이상의 과거 데이터가 필요하며, 통신처럼 안정적인 산업은 3년으로도 충분합니다.

## 3표 통합 모델 구축 프로세스

### Step 1: 과거 데이터 정리

5개년 재무 데이터를 표준화합니다:

| 항목 | 처리 방법 |
|------|----------|
| 손익계산서 | 공통형(매출=100%) 분석 |
| 재무상태표 | 총자산 대비 비중 분석 |
| 현금흐름표 | 주요 항목 추출 |
| 비율 분석 | Layer 2 지표 산출 |
| 일회성 항목 | 정상화(Normalization) |

#### 과거 데이터 정리 심화

**정상화 기법**: 과거 실적에서 반복되지 않는 항목을 식별하고 제거하여 핵심 영업 실적을 파악합니다.

| 비경상 항목 유형 | 구체적 사례 | 처리 방법 |
|-----------------|-----------|----------|
| 자산 처분 손익 | 부동산 매각 차익, 투자 유가증권 처분 | 영업이익에서 제거 |
| 구조조정 비용 | 희망퇴직 비용, 공장 폐쇄 비용 | 정상 비용에서 제외 |
| 소송 관련 손익 | 특허 소송 배상금, 합의금 수취 | 일회성으로 분류 |
| 회계 기준 변경 | K-IFRS 전환 효과, 리스 기준 변경 | 소급 조정하여 일관성 확보 |
| 대손 관련 | 대규모 대손 상각, 대손 환입 | 정상 수준으로 조정 |

**세그먼트 데이터 추출**: DART 사업보고서의 부문별 정보를 활용하여 사업부별 매출과 영업이익을 분리합니다.

```
전체 매출 분해 예시 (다각화 기업):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
사업부 A: 매출 5,000억 원 (OPM 15%)
사업부 B: 매출 3,000억 원 (OPM 8%)
사업부 C: 매출 1,500억 원 (OPM -5%) ← 신규 사업
내부 거래 제거: -500억 원
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
연결 매출: 9,000억 원 (OPM 10%)
```

세그먼트별로 분리하면 각 사업부의 성장률과 수익성을 독립적으로 추정할 수 있어 모델의 정확도가 크게 향상됩니다.

### Step 2: 핵심 가정 수립

**손익계산서 가정**

| 항목 | 가정 방법 | 근거 |
|------|----------|------|
| 매출 성장률 | Top-down + Bottom-up | 시장 규모 × 점유율 × ASP |
| 매출원가율 | 과거 추세 + 원가 전망 | 원재료비, 인건비 변동 |
| 판관비율 | 과거 추세 + 전략 | R&D, 마케팅 계획 |
| 영업외 수익/비용 | 보수적 가정 | 이자비용은 부채 수준에 연동 |
| 법인세율 | 실효세율 적용 | 과거 실효세율 or 법정세율 |

**재무상태표 가정**

| 항목 | 가정 방법 | 근거 |
|------|----------|------|
| 매출채권 | 회전일수 유지 | 과거 평균 DSO |
| 재고자산 | 회전일수 유지 | 과거 평균 DIO |
| 매입채무 | 회전일수 유지 | 과거 평균 DPO |
| 유형자산 | CAPEX - 감가상각 | 투자 계획, 과거 패턴 |
| 차입금 | 만기 도래 + 신규 차입 | 자금 계획 |

**현금흐름표**: 손익계산서와 재무상태표에서 자동 도출

### Step 3: 3표 연결 구축

```
손익계산서 추정
  → 당기순이익 → 재무상태표 이익잉여금
  → 감가상각비 → 현금흐름표
  ↓
재무상태표 추정
  → 자산/부채 변동 → 현금흐름표 (운전자본, 투자)
  → 차입금 변동 → 이자비용 → 손익계산서
  ↓
현금흐름표 도출
  → 기말 현금 → 재무상태표 현금 (검증)
```

#### 3표 연결 심화: 순환참조 처리

3표 통합 모델에서 가장 까다로운 부분은 **순환참조** 문제입니다. 이자비용이 부채 잔액에 의존하고, 부채 잔액이 현금흐름에 의존하며, 현금흐름이 이자비용에 의존하는 구조가 형성됩니다.

```
순환참조 구조:
이자비용 → 순이익 → 현금흐름 → 기말 현금
    ↑                              ↓
    ←── 차입금 잔액 ←── 자금 과부족 ←┘
```

**해결 방법 1: 반복 계산 활용**
- 엑셀의 반복 계산 기능을 활성화하여 자동으로 수렴시킴
- 최대 반복 횟수 100회, 최대 변동값 0.001로 설정

**해결 방법 2: 순환참조 차단기 도입**
- 별도 셀에 스위치(0 또는 1)를 만들어 순환참조를 제어
- 모델 수정 시 스위치를 끄고, 완료 후 다시 켜는 방식

**밸런싱 플러그 설계**: 재무상태표의 자산과 부채+자본이 반드시 일치하도록 조정 항목을 설정합니다.

| 플러그 유형 | 적용 상황 | 동작 방식 |
|------------|----------|----------|
| 현금 플러그 | 잉여 현금 발생 시 | 초과 현금이 현금 및 현금성 자산에 축적 |
| 차입금 플러그 | 자금 부족 시 | 부족분만큼 단기 차입금(리볼버)이 자동 증가 |

### Step 4: FCF 산출

```
매출액 (추정)
× 영업이익률 (가정)
= 영업이익
× (1 - 세율)
= NOPAT
+ 감가상각비
- CAPEX
- 순운전자본 변동
= FCFF (잉여현금흐름)
```

## 모델링 실전 가이드

### 매출 추정 방법

**방법 1: 물량 × 단가**
```
시장 규모 (TAM)
× 시장 점유율
= 판매 물량
× 평균 판매 단가 (ASP)
= 매출액
```

**방법 2: 세그먼트별 추정**
```
사업부 A 매출 (성장률 적용)
+ 사업부 B 매출 (성장률 적용)
+ 신규 사업 매출
= 총 매출
```

**방법 3: 드라이버 기반**
```
기존 고객 수 × 유지율 × ARPU
+ 신규 고객 수 × ARPU
= 매출액
```

#### 매출 추정 심화: 한국 산업별 사례

산업마다 매출을 구동하는 핵심 변수가 다릅니다. 한국 주요 산업의 매출 추정 프레임워크를 정리합니다.

**반도체: 비트 성장 × ASP**
```
DRAM 매출 추정:
  출하 비트 성장률: +15~25% (산업 수요 성장)
  × ASP 변동률: -10~+30% (수급에 따라 변동)
  = DRAM 매출 성장률

NAND 매출 추정:
  출하 비트 성장률: +25~35%
  × ASP 변동률: -15~+20%
  = NAND 매출 성장률
```

핵심은 **메모리 사이클 판단**입니다. 공급 증설 시점과 수요 성장률의 괴리가 ASP를 결정하며, 이 사이클이 3~4년 주기로 반복됩니다.

**통신: 가입자 × ARPU**
```
무선 매출 추정:
  기존 가입자 수 × (1 - 해지율) = 유지 가입자
  + 신규 가입자 유입
  = 총 가입자 수
  × 월 ARPU × 12개월
  = 무선 서비스 매출

유선/기타:
  초고속인터넷 가입자 × 월 요금
  + IPTV 가입자 × 월 요금
  + 기업향 회선 매출
```

한국 통신 시장은 성숙기로 가입자 순증보다 **ARPU 상승** (5G 전환, 요금제 상향)이 핵심 드라이버입니다.

**건설: 수주잔고 소진**
```
매출 추정:
  기초 수주잔고
  + 당기 신규 수주
  - 당기 매출 인식 (진행률 기반)
  = 기말 수주잔고

진행률 산정:
  실제 투입 원가 / 총 예정 원가 = 공사 진행률
  수주 금액 × 진행률 = 누적 매출 인식
```

건설업은 **수주잔고가 향후 2~3년의 매출을 선행**하므로 수주 추이가 가장 중요한 지표입니다. 신규 수주 추정 시 국내외 발주 물량, 정부 SOC 예산, 해외 프로젝트 파이프라인을 종합적으로 고려합니다.

**이커머스/플랫폼: 거래액 × 수수료율**
```
총 거래액(GMV) 추정:
  활성 구매자 수 × 연간 구매 빈도 × 건당 결제 금액
  = GMV

매출 추정:
  GMV × 판매 수수료율
  + 광고 매출 (광고주 수 × 광고 단가)
  + 물류/풀필먼트 매출
  = 총 매출
```

### 비용 추정 심화

비용을 정확히 추정하려면 **변동비와 고정비를 분리**하는 것이 핵심입니다.

#### 변동비 vs 고정비 모델링

| 비용 유형 | 특성 | 추정 방법 | 대표 항목 |
|----------|------|----------|----------|
| 변동비 | 매출에 비례 | 매출 대비 비율 적용 | 원재료비, 외주가공비, 운반비 |
| 고정비 | 매출과 무관하게 일정 | 절대 금액 or 인플레이션 조정 | 임차료, 기본 인건비, 감가상각비 |
| 준변동비 | 일정 구간까지 고정, 이후 증가 | 단계 함수 적용 | 인력 충원, 물류 센터 운영비 |

**단계 비용 함수 예시**:
```
물류 센터 운영비:
  매출 1,000억 원 이하 → 연간 50억 원 (기존 센터 1개)
  매출 1,001~2,000억 원 → 연간 90억 원 (센터 2개)
  매출 2,001~3,500억 원 → 연간 130억 원 (센터 3개)
```

매출이 특정 임계점을 넘으면 추가 설비 투자나 인력 충원이 필요하여 비용이 단계적으로 증가합니다. 이를 무시하면 고성장 시나리오에서 이익률을 과대 추정하게 됩니다.

#### R&D 자본화 영향

K-IFRS에서는 개발비의 일부를 무형자산으로 자본화할 수 있습니다. 이에 따라 같은 R&D 지출도 회계 처리 방식에 따라 손익이 달라집니다.

| 처리 방식 | 손익 영향 | 현금흐름 영향 |
|----------|----------|-------------|
| 전액 비용 처리 | 당기 영업이익 감소 | 영업활동 현금흐름 감소 |
| 일부 자본화 | 당기 영업이익 상대적 양호, 향후 상각비 발생 | 투자활동 현금흐름 감소 |

모델링 시 기업의 R&D 자본화 비율이 변동하는지 확인하고, 동종 업계 대비 과도하게 자본화하는 기업은 보수적으로 조정합니다.

### CAPEX 추정

| 구분 | 추정 방법 |
|------|----------|
| 유지 CAPEX | ≈ 감가상각비 (현 수준 유지) |
| 성장 CAPEX | 매출 성장에 필요한 추가 투자 |
| 총 CAPEX | 유지 + 성장 CAPEX |

과거 CAPEX/매출 비율, CAPEX/감가상각 비율을 참고

#### CAPEX/감가상각 심화

**자산 스케줄 구축**: 유형자산의 취득, 감가상각, 처분을 체계적으로 추적합니다.

```
유형자산 스케줄:
  기초 장부가액
  + 당기 CAPEX (신규 취득)
  - 당기 감가상각비
  - 자산 처분 (장부가액 기준)
  = 기말 장부가액
```

**유지 CAPEX vs 성장 CAPEX 분리 방법**:

| 판단 기준 | 유지 CAPEX | 성장 CAPEX |
|----------|-----------|-----------|
| 규모 비교 | 감가상각비의 80~120% 수준 | 감가상각비 초과분 |
| 경영진 발언 | "기존 설비 교체" | "신규 라인 증설", "생산능력 확대" |
| 공시 확인 | DART 주석의 자본적 지출 내역 | 사업보고서의 투자 계획 |

**자산 유형별 내용연수 가정**:

| 자산 유형 | 일반적 내용연수 | 비고 |
|----------|---------------|------|
| 건물 | 20~40년 | 구조에 따라 상이 |
| 기계장치 | 5~15년 | 산업별 차이 큼 |
| 반도체 장비 | 5~7년 | 기술 세대 변화 반영 |
| 차량운반구 | 4~6년 | |
| 공구/비품 | 3~5년 | |
| 소프트웨어 | 3~5년 | 무형자산으로 분류 |

### 감가상각 추정

$$\text{감가상각비} = \frac{\text{유형자산 총액}}{\text{평균 내용연수}}$$

또는 과거 감가상각률(감가상각비/기초 유형자산)을 적용

## 운전자본 모델링 상세

운전자본은 일상 영업활동에 필요한 단기 자금으로, 모델링에서 간과하기 쉽지만 현금흐름에 큰 영향을 미칩니다.

### 일수 기반 추정법

운전자본 항목을 매출 또는 매출원가 대비 회전일수로 추정합니다:

```
매출채권 = 매출액 × (DSO / 365)
재고자산 = 매출원가 × (DIO / 365)
매입채무 = 매출원가 × (DPO / 365)

순운전자본(NWC) = 매출채권 + 재고자산 - 매입채무
NWC 변동 = 당기 NWC - 전기 NWC
```

| 항목 | 산식 | 한국 제조업 평균 |
|------|------|----------------|
| DSO | 매출채권 / 일평균 매출 | 50~70일 |
| DIO | 재고자산 / 일평균 매출원가 | 40~80일 |
| DPO | 매입채무 / 일평균 매출원가 | 30~60일 |
| CCC | DSO + DIO - DPO | 40~90일 |

### 계절성 조정

분기 모델에서는 매출의 계절성에 따라 운전자본이 크게 변동합니다:

- **유통업**: 4분기(연말 세일) 매출 집중 → 3분기 말 재고 축적
- **건설업**: 1분기(동절기) 매출 감소 → 매출채권 회수 지연
- **전자제품**: 하반기 신제품 출시 → 2분기 부품 재고 확대

연간 모델에서는 평균 회전일수를 사용하되, 기말 시점의 계절적 왜곡을 고려하여 평균 잔액을 사용하는 것이 더 정확합니다.

### 마이너스 운전자본

일부 비즈니스 모델은 구조적으로 마이너스 운전자본을 가집니다:

| 유형 | 사례 | 원인 |
|------|------|------|
| 선수금 모델 | 온라인 게임, 구독 서비스 | 대금을 먼저 수취하고 서비스를 나중에 제공 |
| 유통 파워 | 대형 마트, 이커머스 플랫폼 | 매입채무 지급 기한 > 매출채권 회수 기한 |
| 역진행 매출 | 상품권, 선불카드 | 선수수익이 부채에 계상 |

마이너스 운전자본 기업은 매출 성장 시 운전자본이 오히려 현금을 창출하므로, FCFF 산출에서 운전자본 변동이 양(+)의 효과를 냅니다. 이러한 특성은 기업의 본질적 현금 창출력을 높여주는 구조적 장점입니다.

## 한국 기업 모델링 특수 고려

### 연결 vs 별도 모델

| 구분 | 용도 |
|------|------|
| **연결 모델** | 그룹 전체 가치 산출 (표준) |
| **별도 모델** | 해당 법인 고유 가치, 지주회사 NAV |
| **세그먼트 모델** | 사업부별 SOTP 밸류에이션 |

### 한국 기업 특수 항목

1. **환율 가정**: 수출 비중 높은 기업은 환율 시나리오 필수
2. **반도체 사이클**: ASP와 물량의 사이클 반영
3. **관계사 거래**: 내부 거래 제거 (연결) 또는 지속성 가정 (별도)
4. **K-IFRS 영향**: 수익 인식 기준, 리스 회계 반영

### 한국 기업 모델링 심화

#### 수출 기업의 환율 모델링

수출 비중이 높은 한국 기업은 원/달러 환율이 매출과 영업이익에 직접적인 영향을 미칩니다.

```
환율 영향 모델링 구조:
  달러 매출 (USD) × 원/달러 환율 = 원화 매출
  달러 원가 (USD) × 원/달러 환율 = 원화 원가

  환율 민감도:
  원/달러 10원 상승 시
  → 매출 변동: + △ (수출 비중에 비례)
  → 원가 변동: + △ (수입 원자재 비중에 비례)
  → 영업이익 변동: 순효과 = 매출 효과 - 원가 효과
```

**환율 가정 수립 시 참고 사항**:
- 컨센서스 환율 전망 활용 (블룸버그, 증권사 리서치)
- 기업의 환헷지 비율 확인 (사업보고서 주석)
- 자연적 헷지 효과 반영 (달러 매출 vs 달러 원가 상쇄)

#### 반도체 사이클 오버레이

반도체 기업 모델링에서는 **업황 사이클을 별도 레이어로 반영**합니다:

```
기본 추정 (구조적 성장):
  비트 출하량: 연간 +15~20% (장기 수요 성장)
  ASP: 장기 하락 추세 (연간 -5~10%)

사이클 오버레이:
  업사이클 (공급 부족): ASP +20~50% 추가 상승
  다운사이클 (공급 과잉): ASP -30~50% 추가 하락

적용 방법:
  추정 연도의 사이클 위치 판단
  → 업황 조정 계수를 기본 ASP에 적용
  → 영업이익률 변동 범위: 5~50% (사이클에 따라 극단적 변동)
```

반도체 사이클 판단을 위한 선행 지표:
- 재고 수준 (업계 평균 재고주간 추이)
- 설비투자 발표 (경쟁사 증설 계획)
- 수요 측 지표 (서버 출하량, 스마트폰 판매량, 데이터센터 투자)

#### 건설업 진행률 기반 모델

건설업은 K-IFRS 수익 인식 기준에 따라 **공사 진행률 기반**으로 매출을 인식합니다:

```
프로젝트별 모델링:
  프로젝트 A: 총 수주액 5,000억 원, 공사기간 3년
    1차년도: 진행률 30% → 매출 1,500억 원
    2차년도: 진행률 45% → 매출 2,250억 원
    3차년도: 진행률 25% → 매출 1,250억 원

  프로젝트 B: 총 수주액 3,000억 원, 공사기간 2년
    ...
```

**공사손실충당부채**: 예정 원가가 수주액을 초과하는 프로젝트는 손실이 예상되는 시점에 일시에 인식해야 합니다. DART 주석에서 공사손실충당부채 규모를 확인하여 모델에 반영합니다.

### DART 데이터 활용

1. 사업보고서: 5개년 요약 재무정보 (기초 데이터)
2. 분기보고서: 분기별 세그먼트 매출/영업이익
3. 주석: 감가상각 내역, 차입금 만기 구조
4. 사업 내용: 투자 계획, 시설 현황

## 모델 검증 방법

모델 완성 후에는 반드시 체계적인 검증을 수행해야 합니다. 오류가 있는 모델에 기반한 밸류에이션은 투자 손실로 직결됩니다.

### 오류 체크 기법

**자동 검증 항목**: 모델에 내장하여 실시간으로 확인합니다.

| 검증 항목 | 산식 | 정상 범위 |
|----------|------|----------|
| 재무상태표 밸런스 | 자산 - (부채 + 자본) | 정확히 0 |
| 현금 검증 | 현금흐름표 기말 현금 - 재무상태표 현금 | 정확히 0 |
| 이익잉여금 검증 | 기초 이익잉여금 + 순이익 - 배당 = 기말 이익잉여금 | 정확히 0 |
| 감가상각 누적 | 누적 감가상각 ≤ 취득원가 | 항상 참 |

### 합리성 테스트

추정 결과가 현실적인 범위에 있는지 점검합니다:

| 점검 항목 | 경고 신호 |
|----------|----------|
| 영업이익률 | 과거 최고치를 지속적으로 초과 |
| 매출 성장률 | 산업 성장률 대비 비현실적으로 높음 |
| CAPEX/매출 | 과거 평균 대비 급격한 변동 |
| 운전자본/매출 | 과거 추세와 크게 괴리 |
| 부채비율 | 재무 안정성 훼손 수준으로 상승 |
| ROIC | 산업 평균 대비 지나치게 높은 수준이 영구 지속 |

### 과거 데이터 역검증

모델의 신뢰성을 높이기 위해 **과거 실적에 대한 역검증**을 실시합니다:

1. 과거 3년 중 2년의 데이터로 모델 구축
2. 나머지 1년의 실적을 모델로 추정
3. 추정치와 실제 실적의 괴리율 분석
4. 괴리가 큰 항목의 가정을 재검토

괴리율이 매출 ±5%, 영업이익 ±10% 이내라면 모델의 기본 구조가 적절한 것으로 판단합니다. 이 범위를 벗어나면 핵심 드라이버의 가정을 재점검해야 합니다.

### 모델링 체크리스트

- [ ] 과거 5년 재무 데이터를 정리했는가?
- [ ] 비경상 항목을 식별하고 정상화했는가?
- [ ] 세그먼트별 매출/이익을 분리했는가?
- [ ] 핵심 가정(매출 성장률, 마진, CAPEX)이 합리적인가?
- [ ] 변동비와 고정비를 구분하여 추정했는가?
- [ ] 3표가 올바르게 연결되어 있는가?
- [ ] 순환참조가 적절히 처리되었는가?
- [ ] 재무상태표가 밸런스하는가? (자산 = 부채 + 자본)
- [ ] 현금흐름표의 기말 현금 = 재무상태표 현금인가?
- [ ] 운전자본 회전일수가 합리적 범위인가?
- [ ] CAPEX/감가상각 스케줄이 일관적인가?
- [ ] 시나리오별 민감도 분석을 수행했는가?
- [ ] 과거 데이터 역검증을 실시했는가?
- [ ] 가정의 출처와 근거를 문서화했는가?

### 모델링 → DCF 연결

```
재무 모델 (3표 추정)
  → FCFF 산출 (5~10년)
  → 터미널 밸류 산출
  → WACC으로 할인
  → 기업가치(EV)
  → 순차입금 차감
  → 주주가치(Equity Value)
  → 주당 가치
  → 현재 주가와 비교
  → 투자 판단
```
