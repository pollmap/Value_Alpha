# Contributing to 금융 위키

금융 위키에 기여해 주셔서 감사합니다! 아래 가이드라인을 따라주세요.

## 기여 방법

### 1. Issue 먼저 확인
- 새로운 기능 제안이나 버그 수정 전에 기존 Issue를 확인하세요
- 중복 Issue를 피하고, 관련 Issue가 있다면 댓글로 참여하세요

### 2. Fork & Branch
```bash
# 저장소 Fork 후 클론
git clone https://github.com/YOUR_USERNAME/Value_Alpha.git

# 브랜치 생성 (기능별로 구분)
git checkout -b feature/your-feature-name
```

### 3. 개발 환경 설정
```bash
cd wiki
npm install
npm run start
```

### 4. 코드 작성 규칙

#### 파일 구조
- 문서: `wiki/docs/` 디렉토리
- 컴포넌트: `wiki/src/components/`
- 스타일: `wiki/src/css/`

#### 커밋 메시지
```
type(scope): 간단한 설명

- feat: 새로운 기능
- fix: 버그 수정
- docs: 문서 수정
- style: 포맷팅 (코드 변경 X)
- refactor: 리팩토링
- test: 테스트
- chore: 기타
```

### 5. Pull Request
1. 변경사항 push
2. PR 생성 시 템플릿 따르기
3. 리뷰어 지정 대기
4. 피드백 반영 후 머지

## 금지 사항

다음 내용이 포함된 PR은 거부됩니다:

- 악의적인 코드 (악성 스크립트, 백도어 등)
- 개인정보 (실명, 연락처, 주소 등)
- 민감한 정보 (API 키, 비밀번호)
- 저작권 침해 콘텐츠
- 스팸, 광고, 홍보 링크
- 정치적/종교적 콘텐츠
- 부정확한 금융 정보 (검증되지 않은 투자 조언)

## 콘텐츠 가이드라인

### 금융 정보 작성 시
1. **출처 명시**: 통계, 수치는 반드시 출처 명시
2. **객관성 유지**: 특정 종목/상품 추천 금지
3. **면책 조항**: 투자 관련 내용에는 면책 조항 포함
4. **최신 정보**: 법규, 세율 등은 최신 정보로 업데이트

### 문서 스타일
- 한국어로 작성 (전문 용어는 영문 병기 가능)
- 명확하고 간결한 문장 사용
- 코드 블록에는 적절한 언어 표시
- 수식은 KaTeX 문법 사용

## 리뷰 프로세스

1. **자동 검사**: CI/CD 파이프라인 통과 필수
2. **코드 리뷰**: 최소 1명의 maintainer 승인 필요
3. **내용 검토**: 금융 정보의 정확성 검증
4. **머지**: squash merge로 깔끔한 히스토리 유지

## 질문이 있으시면

- GitHub Discussions에서 질문해 주세요
- Issue로 버그 리포트/기능 요청해 주세요

감사합니다! 🙏
